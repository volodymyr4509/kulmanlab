# Undo

يعكس أمر `undo` آخر تغيير على الرسم — خطوة واحدة في كل استدعاء. يُسجَّل كل إضافة أو حذف أو تحرير للعناصر كإدخال سجل منفصل. التراجع يتراجع عبر هذه الإدخالات بالترتيب العكسي.

## كيفية التراجع

- اكتب `undo` في المحطة، أو
- انقر على زر **Undo** في شريط الأدوات.

كل استدعاء يعكس إجراءً مسجَّلاً واحداً. استدعِه مراراً للتراجع أكثر.

## سلوك السجل

| التفصيل | القيمة |
|---------|--------|
| الخطوات لكل ملف | حتى **20** |
| التخزين | المتصفح (IndexedDB / localStorage)، لكل اسم ملف |
| يستمر عبر إعادة التحميل | نعم — يُستعاد السجل عند إعادة فتح الملف |
| إجراء جديد بعد تراجع | يمسح جميع إدخالات الإعادة الأمامية من الموضع الحالي |
| أقدم إدخال عند الامتلاء | يُحذف لإفساح مجال للتغيير الأحدث |

يُسجَّل كل تحوير للعنصر: رسم عناصر جديدة وحذفها وتحرير نقاط نهايتها بنقاط التحكم وتطبيق Move وRotate وScale وMirror وTrim وExtend وOffset — كلها تُنشئ إدخالات سجل.

## التراجع مقابل الإعادة

| | التراجع | الإعادة |
|---|------|------|
| الاتجاه | يتقدم **للخلف** عبر السجل | يتقدم **للأمام** عبر الإدخالات المُتراجَع عنها |
| متاح عندما | يوجد إجراء مسجَّل واحد على الأقل | تُنفِّذ على الأقل تراجعاً واحداً ولم تُجرِ أي إجراء جديد |
| يُمسح بواسطة | لا شيء — يتراكم السجل حتى حد 20 خطوة | أي إجراء رسم جديد |

استخدم [Redo](../redo/) لإعادة تطبيق إجراء مُتراجَع عنه. أزرار شريط الأدوات تكون معتمة عندما يكون الاتجاه المعني غير متاح.
