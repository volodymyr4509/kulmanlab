# Import

Η εντολή **Import** φορτώνει υπάρχον σχέδιο από το τοπικό σύστημα αρχείων σας στο KulmanLab CAD. Υποστηρίζονται τόσο η τυπική μορφή **DXF** όσο και η ιδιόκτητη μορφή **JSON** του KulmanLab.

## Πώς να εισάγετε αρχείο

1. Κάντε κλικ στο κουμπί **Import** της γραμμής εργαλείων (εικονίδιο φακέλου) στο πάνελ File στο πάνω μέρος της οθόνης.
2. Ανοίγει η επιλογή αρχείου του browser σας. Πλοηγηθείτε στο αρχείο σχεδίου σας και επιλέξτε το.
3. Το σχέδιο φορτώνεται αμέσως στον καμβά. Η προβολή προσαρμόζεται αυτόματα σε όλες τις οντότητες.

Εναλλακτικά, μπορείτε να σύρετε και να αφήσετε αρχείο απευθείας στον καμβά.

## Υποστηριζόμενες μορφές αρχείων

| Μορφή | Επέκταση | Πότε να χρησιμοποιείτε |
|-------|----------|----------------------|
| **DXF** | `.dxf` | Σχέδια από FreeCAD, LibreCAD ή άλλα εργαλεία CAD |
| **JSON** *(native)* | `.json` | Σχέδια προηγουμένως αποθηκευμένα από KulmanLab CAD — πλήρης πιστότητα |

## Τι εισάγεται από DXF

Το KulmanLab αναλύει τους ακόλουθους τύπους οντοτήτων DXF:

| Τύπος οντότητας | Κώδικας DXF | Σημειώσεις |
|----------------|-------------|-----------|
| Line | `LINE` | |
| Circle | `CIRCLE` | |
| Arc | `ARC` | |
| Ellipse | `ELLIPSE` | |
| Polyline | `LWPOLYLINE` | |
| Spline | `SPLINE` | |
| Text | `TEXT`, `MTEXT` | |
| Dimension | `DIMENSION` | |
| Multileader | `MULTILEADER` | |

Οι ορισμοί επιπέδων και οι πίνακες linetype εισάγονται επίσης από το αρχείο DXF όταν υπάρχουν.

Οντότητες που χρησιμοποιούν μη υποστηριζόμενους τύπους DXF παραλείπονται σιωπηλά — το υπόλοιπο σχέδιο φορτώνεται κανονικά.

## Ονομασία αρχείων και αποθήκευση

Όταν εισάγετε αρχείο, μια χρονική σήμανση προσαρτάται στο όνομα αρχείου (για παράδειγμα `myplan_May22_14:30:00.dxf`). Αυτό σας επιτρέπει να διατηρείτε πολλαπλές εκδόσεις του ίδιου σχεδίου στα [Πρόσφατα Αρχεία](../files/) χωρίς συγκρούσεις ονομάτων. Αν το όνομα αρχείου ήδη περιέχει χρονική σήμανση, χρησιμοποιείται ως έχει.

Το σχέδιο αποθηκεύεται αυτόματα στην αποθήκευση browser (IndexedDB) μετά την εισαγωγή, οπότε εμφανίζεται στο πάνελ [Files](../files/) και επιβιώνει από επαναφορτώσεις σελίδας.

## Τι συμβαίνει στο τρέχον σχέδιο

Η εισαγωγή αντικαθιστά τον τρέχοντα καμβά. Δεν υπάρχει συγχώνευση ή προσάρτηση. Αν έχετε μη αποθηκευμένες αλλαγές, [εξάγετε](../export/) το τρέχον σχέδιο πρώτα.

## Κατά την εκκίνηση

Το KulmanLab ανοίγει αυτόματα το πιο πρόσφατα επεξεργασμένο αρχείο κατά τη φόρτωση σελίδας. Αν δεν υπάρχουν αποθηκευμένα αρχεία, φορτώνεται ένα προεπιλεγμένο δείγμα σχεδίου.

## Αντιμετώπιση προβλημάτων

| Πρόβλημα | Πιθανή αιτία | Λύση |
|----------|-------------|------|
| Ο καμβάς είναι κενός μετά την εισαγωγή | Οντότητες DXF χρησιμοποιούν μη υποστηριζόμενους τύπους (π.χ. HATCH, INSERT) | Οι οντότητες παραλείφθηκαν — ελέγξτε για μήνυμα "no entities found" στο τερματικό |
| Το κουμπί Import δεν κάνει τίποτα | Ο browser απέκλεισε την επιλογή αρχείου | Κάντε ξανά κλικ στο κουμπί· ορισμένοι browsers απαιτούν νέα χειρονομία χρήστη |
| Οι διαστάσεις φαίνονται λανθασμένες | DXF από εργαλείο που γράφει μη τυπική γεωμετρία διάστασης | Επανεξάγετε από την εφαρμογή πηγής χρησιμοποιώντας τρέχουσα έκδοση DXF |
