# Undo

Η εντολή `undo` αναιρεί την τελευταία αλλαγή στο σχέδιο — ένα βήμα ανά κλήση. Κάθε προσθήκη, διαγραφή ή επεξεργασία οντοτήτων καταγράφεται ως ξεχωριστή εγγραφή ιστορικού. Το Undo κάνει βήματα πίσω μέσα σε αυτές τις εγγραφές με αντίστροφη σειρά.

## Πώς να κάνετε undo

- Πληκτρολογήστε `undo` στο τερματικό, ή
- Κάντε κλικ στο κουμπί **Undo** της γραμμής εργαλείων.

Κάθε κλήση αναιρεί μια καταγεγραμμένη ενέργεια. Καλέστε την επανειλημμένα για να κάνετε βήματα πιο πίσω.

## Συμπεριφορά ιστορικού

| Λεπτομέρεια | Τιμή |
|------------|------|
| Βήματα ανά αρχείο | Έως **20** |
| Αποθήκευση | Browser (IndexedDB / localStorage), ανά όνομα αρχείου |
| Επιβιώνει επαναφόρτωσης σελίδας | Ναι — το ιστορικό αποκαθίσταται όταν ανοίγετε ξανά το αρχείο |
| Νέα ενέργεια μετά από undo | Καθαρίζει όλες τις εγγραφές redo μπροστά από την τρέχουσα θέση |
| Παλαιότερη εγγραφή όταν γεμίσει | Απορρίπτεται για να κάνει χώρο για τη νεότερη αλλαγή |

Κάθε μετάλλαξη οντότητας καταγράφεται: σχεδίαση νέων οντοτήτων, διαγραφή οντοτήτων, επεξεργασία grip τελικών σημείων, εφαρμογή Move, Rotate, Scale, Mirror, Trim, Extend και Offset δημιουργούν εγγραφές ιστορικού.

## Undo vs Redo

| | Undo | Redo |
|---|------|------|
| Κατεύθυνση | Βήματα **πίσω** στο ιστορικό | Βήματα **μπροστά** μέσα σε αναιρεθείσες εγγραφές |
| Διαθέσιμο όταν | Υπάρχει τουλάχιστον μία καταγεγραμμένη ενέργεια | Έχει εκτελεστεί τουλάχιστον ένα Undo και δεν έχει γίνει νέα ενέργεια |
| Διαγράφεται από | Τίποτα — το ιστορικό συσσωρεύεται μέχρι το όριο 20 βημάτων | Οποιαδήποτε νέα ενέργεια σχεδίασης |

Χρησιμοποιήστε [Redo](./redo) για επανεφαρμογή αναιρεθείσας ενέργειας. Τα κουμπιά γραμμής εργαλείων είναι γκρι όταν η αντίστοιχη κατεύθυνση δεν είναι διαθέσιμη.
