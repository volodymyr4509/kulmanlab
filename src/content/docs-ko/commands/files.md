# Files

**Files** 명령어는 **최근 파일** 패널을 엽니다 — 브라우저의 로컬 저장소에 저장된 모든 도면의 목록입니다. 이전 도면을 다시 열거나 더 이상 필요 없는 파일을 제거하는 데 사용합니다.

## 최근 파일 패널 여는 방법

화면 상단 파일 패널의 **Files** 도구 모음 버튼(기록 아이콘)을 클릭합니다. 패널이 캔버스 왼쪽에 열립니다.

## 최근 파일 패널

목록의 각 항목에는 파일 이름과 두 개의 작업 버튼이 표시됩니다:

| 버튼 | 동작 |
|------|------|
| ✓ **복원** | 도면을 캔버스에 로드합니다 — 현재 내용을 대체합니다 |
| ✕ **삭제** | 브라우저 저장소에서 파일을 영구적으로 제거합니다 |

저장된 파일이 없으면 패널에 "저장된 파일 없음"이 표시됩니다.

## 파일 저장 방법

KulmanLab은 도면을 **IndexedDB** — 지속적인 브라우저 내 데이터베이스에 저장합니다. 즉:

- 파일은 페이지 새로고침 및 브라우저 재시작 후에도 유지됩니다.
- 파일은 **기기에 로컬로 저장**됩니다 — 서버에 업로드되지 않습니다.
- 각 브라우저와 기기는 독립적인 저장소를 가집니다. 한 컴퓨터의 Chrome에 저장된 도면은 Firefox에서나 다른 기기에서 사용할 수 없습니다.
- 브라우저의 사이트 데이터를 지우면 저장된 모든 파일이 삭제됩니다.

## 시작 시 자동 파일 로드

KulmanLab CAD를 열면 앱이 자동으로 저장소에서 **가장 최근에 수정된 파일**을 로드합니다. 매번 파일 패널에서 수동으로 열 필요가 없습니다.

## 저장소 관리

저장할 수 있는 도면 수에는 고정된 제한이 없지만 브라우저 저장소는 유한합니다. 저장소 경고가 표시되면 파일 패널에서 오래된 파일을 삭제하세요.

한 번에 저장된 모든 도면을 제거하려면 [Wipe Storage](../wipestorage/) 명령어를 사용하세요.

## 파일 이름 및 버전 관리

각 파일의 이름에는 타임스탬프가 있습니다(예: `myplan_May22_14:30:00.dxf`). 이 타임스탬프는 파일을 [Import](../import/)할 때 자동으로 추가되어, 같은 소스 파일을 다시 가져오면 이전 버전을 덮어쓰지 않고 새 항목이 만들어집니다.

## 관련 명령어

- [Import](../import/) — 파일 시스템에서 브라우저 저장소로 도면 불러오기
- [Export](../export/) — 도면을 파일 시스템으로 다운로드
- [New File](../new-file/) — 빈 도면 시작(자동으로 저장됨)
- [Wipe Storage](../wipestorage/) — 브라우저 저장소에서 저장된 모든 파일 지우기
