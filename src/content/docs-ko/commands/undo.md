# Undo

`undo` 명령어는 도면의 마지막 변경 사항을 되돌립니다 — 호출당 한 단계씩. 객체의 각 추가, 삭제 또는 편집은 별도의 기록 항목으로 기록됩니다. 실행 취소는 이러한 항목을 역순으로 단계별로 이동합니다.

## 실행 취소 방법

- 터미널에 `undo`를 입력하거나,
- **Undo** 도구 모음 버튼을 클릭합니다.

각 호출은 기록된 하나의 작업을 되돌립니다. 더 이전으로 단계를 진행하려면 반복적으로 호출합니다.

## 기록 동작

| 세부 정보 | 값 |
|----------|-----|
| 파일당 단계 | 최대 **20** |
| 저장 위치 | 브라우저(IndexedDB / localStorage), 파일 이름별 |
| 페이지 새로고침 후 유지 | 예 — 파일을 다시 열면 기록이 복원됨 |
| 실행 취소 후 새 작업 | 현재 위치 이후의 모든 다시 실행 항목을 지움 |
| 꽉 차면 가장 오래된 항목 | 최신 변경 사항을 위한 공간 확보를 위해 삭제됨 |

모든 객체 변경 사항이 기록됩니다: 새 객체 그리기, 객체 삭제, 그립 편집 끝점, 이동, 회전, 축척, 대칭, 자르기, 연장, 간격띄우기 적용 모두 기록 항목을 만듭니다.

## 실행 취소 vs 다시 실행

| | 실행 취소 | 다시 실행 |
|--|----------|----------|
| 방향 | 기록을 **역방향**으로 이동 | 취소된 항목을 **순방향**으로 이동 |
| 사용 가능한 시기 | 기록된 작업이 하나 이상 있을 때 | 실행 취소가 하나 이상 수행되고 새 작업이 수행되지 않았을 때 |
| 지워지는 시기 | 없음 — 기록이 20단계 제한까지 누적됨 | 새 도면 작업이 수행될 때 |

[Redo](../redo/)을 사용하여 취소된 작업을 재적용합니다. 해당 방향을 사용할 수 없을 때 도구 모음 버튼이 회색으로 표시됩니다.
