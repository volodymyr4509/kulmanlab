# 다시 실행

`redo` 명령어는 [실행 취소](./undo)로 되돌린 작업을 재적용하여 실행 취소 기록을 앞으로 이동합니다. 다시 실행은 실행 취소로 뒤로 이동하고 새 변경 사항을 아직 만들지 않은 경우에만 사용 가능합니다.

## 다시 실행 방법

- 터미널에 `redo`를 입력하거나,
- **다시 실행** 도구 모음 버튼을 클릭합니다.

각 호출은 이전에 취소된 하나의 작업을 재적용합니다. 사용 가능한 모든 다시 실행 항목을 통해 앞으로 이동하려면 반복적으로 호출합니다.

## 다시 실행 스택 동작

| 세부 정보 | 동작 |
|----------|------|
| 이후 사용 가능 | 하나 이상의 [실행 취소](./undo) 단계 |
| 지워지는 시기 | **새 도면 작업** — 객체 추가, 편집 또는 삭제 |
| 저장 위치 | 브라우저, 파일별 — 페이지 새로고침 후에도 유지됨(새로고침 전 새 작업이 없었다면) |
| 최대 깊이 | 최대 20개 항목(실행 취소와 동일한 풀) |

새 객체가 그려지거나, 삭제되거나, 수정되면 다시 실행 스택이 지워지고 해당 항목을 복구할 수 없습니다. 새 작업에 의해 대체되지 않은 취소된 작업만 다시 실행할 수 있습니다.

## 다시 실행 vs 실행 취소

| | 다시 실행 | 실행 취소 |
|--|----------|----------|
| 방향 | 취소된 항목을 **순방향**으로 이동 | 기록을 **역방향**으로 이동 |
| 사용 가능한 시기 | 실행 취소 후 새 작업 없을 때 | 기록된 작업이 하나 이상 있을 때 |
| 지워지는 시기 | 새 도면 작업이 수행될 때 | 없음 |

다시 실행할 항목이 없을 때 도구 모음 다시 실행 버튼이 회색으로 표시됩니다. 다시 실행 항목을 만들려면 먼저 [실행 취소](./undo)를 사용하세요.
