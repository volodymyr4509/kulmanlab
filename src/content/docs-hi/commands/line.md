# Line

`line` कमांड व्यक्तिगत सीधे-रेखा खंड खींचती है जो DXF मॉडल में अलग `LINE` ऑब्जेक्ट के रूप में संग्रहीत होते हैं। प्रत्येक खंड के बाद कमांड सक्रिय रहती है और एंडपॉइंट को नए स्टार्ट पॉइंट के रूप में पुनः उपयोग करती है, इसलिए आप एक-एक खंड बनाकर जुड़े पथ बना सकते हैं। [Polyline](../polyline/) के विपरीत, श्रृंखलाबद्ध रेखाएं स्वतंत्र ऑब्जेक्ट बनी रहती हैं — प्रत्येक को अपने पड़ोसियों को प्रभावित किए बिना ट्रिम, एक्सटेंड या डिलीट किया जा सकता है।

## रेखाएं बनाना

1. टर्मिनल में `line` टाइप करें या **Line** टूलबार बटन पर क्लिक करें।
2. **स्टार्ट पॉइंट पर क्लिक करें**, या सटीक निर्देशांक के लिए `X,Y` टाइप करें और **Enter** दबाएं।
3. **एंड पॉइंट पर क्लिक करें** — खंड रखा जाता है और एंडपॉइंट अगला स्टार्ट पॉइंट बन जाता है। यहाँ भी निर्देशांक प्रविष्टि काम करती है।
4. अधिक खंड जोड़ने के लिए क्लिक (या टाइप) करते रहें।
5. रोकने के लिए **Enter** या **Escape** दबाएं।

```
  ●──────────●──────────●──────────●
 start     2nd click  3rd click   Enter to finish
            (auto-becomes next start)
```

केवल एक खंड चाहिए? चरण 3 के तुरंत बाद **Enter** या **Escape** दबाएं।

## निर्देशांक प्रविष्टि

क्लिक करने के बजाय, स्टार्ट या किसी भी बाद के बिंदु के लिए सटीक स्थिति टाइप करें:

1. X मान टाइप करें (अंक, `.`, या `-`)।
2. `,` दबाएं — टर्मिनल `[X], [Y{cursor}]` दिखाता है।
3. Y मान टाइप करें।
4. बिंदु रखने के लिए **Enter** दबाएं।

## कोण लॉकिंग और सटीक लंबाई इनपुट

किसी बिंदु के बाद कर्सर घुमाते समय, कमांड 45° स्नैप अक्ष (0°, 45°, 90°, 135°, …) देखती है। कोण **लॉक** होता है जब:

- कर्सर एंकर से कम से कम **5 × ग्रिप साइज़** दूर हो, **और**
- यह निकटतम अक्ष से लंबवत दूरी के **1 ग्रिप साइज़** के अंदर हो।

लॉक होने पर प्रीव्यू अक्ष पर स्नैप हो जाता है और आप सटीक लंबाई दर्ज कर सकते हैं:

| Key | Action |
|-----|--------|
| `0`–`9`, `.` | लंबाई मान में अंक जोड़ें |
| `-` | नकारात्मक लंबाई — अक्ष के साथ दिशा उलटता है (केवल पहला अक्षर) |
| `Backspace` | अंतिम टाइप किया गया अक्षर हटाएं |
| `Enter` | टाइप की गई दूरी पर एंडपॉइंट रखें |

संचित मान टर्मिनल में लाइव दिखाया जाता है (जैसे `click end point or enter length: 12.5`)। लॉक होने पर क्लिक करने से क्लिक अक्ष पर प्रोजेक्ट होता है, इसलिए एंडपॉइंट हमेशा उस पर पड़ता है।

एंकर बिंदु के पास वापस जाने से लॉक खुल जाती है।

## कीबोर्ड संदर्भ

| Key | Action |
|-----|--------|
| `0`–`9`, `.`, `-` | X निर्देशांक प्रविष्टि शुरू करें, या कोण-लॉक होने पर दूरी |
| `,` | X लॉक करें और Y प्रविष्टि पर जाएं |
| `Backspace` | अंतिम टाइप किया गया अक्षर हटाएं |
| `Enter` | टाइप किए गए निर्देशांक या लंबाई की पुष्टि करें, या कुछ टाइप न हो तो श्रृंखला समाप्त करें |
| `Escape` | श्रृंखला समाप्त करें और बाहर निकलें |

## ग्रिप संपादन — एंडपॉइंट स्ट्रेच करना

एक चयनित रेखा में तीन ग्रिप होते हैं:

| ग्रिप | स्थान | क्या करता है |
|------|-------|--------------|
| **Start** | पहला एंडपॉइंट | ड्रैग करके पुनःस्थित करें — अंत स्थिर रहता है |
| **Midpoint** | रेखा का केंद्र | पूरी रेखा के लिए **Move** सक्रिय करता है |
| **End** | दूसरा एंडपॉइंट | ड्रैग करके पुनःस्थित करें — शुरुआत स्थिर रहती है |

एक एंडपॉइंट को स्ट्रेच करने से दूसरे पर कोई प्रभाव नहीं पड़ता। यह [Polyline](../polyline/) ग्रिप संपादन से अलग है, जहां एक वर्टेक्स को मूव करने से पूरा पथ फिर से आकार लेता है।

## रेखाएं चुनना

| विधि | व्यवहार |
|--------|-----------|
| **क्लिक** | यदि क्लिक खंड की हिट-टेस्ट दूरी के अंदर है तो रेखा चुनता है |
| **दाईं ओर ड्रैग** (सख्त) | रेखा केवल तभी चुनी जाती है जब दोनों एंडपॉइंट बॉक्स के अंदर हों |
| **बाईं ओर ड्रैग** (क्रॉसिंग) | खंड का कोई भी भाग बॉक्स की सीमा पार करता है तो रेखा चुनी जाती है |

## समर्थित संपादन कमांड

रेखाएं **एकमात्र** ऑब्जेक्ट हैं जिन पर [Trim](../trim/) और [Extend](../extend/) काम करती हैं। सभी मानक परिवर्तन कमांड भी लागू होती हैं:

| कमांड | रेखा पर क्या होता है |
|---------|------------------------|
| [Move](../move/) | दोनों एंडपॉइंट को समान विस्थापन से अनुवादित करता है |
| [Copy](../copy/) | नई स्थिति पर समान रेखा बनाता है |
| [Rotate](../rotate/) | दोनों एंडपॉइंट को चुने हुए आधार बिंदु के चारों ओर घुमाता है |
| [Mirror](../mirror/) | दोनों एंडपॉइंट को मिरर अक्ष पर परावर्तित करता है |
| [Scale](../scale/) | दोनों एंडपॉइंट को आधार बिंदु से समान रूप से स्केल करता है |
| [Offset](../offset/) | निश्चित लंबवत दूरी पर समानांतर रेखा बनाता है |
| [Trim](../trim/) | प्रतिच्छेदन पर रेखा काटता है — **केवल रेखाएं** |
| [Extend](../extend/) | निकटतम एंडपॉइंट को सीमा तक खींचता है — **केवल रेखाएं** |
| [Delete](../delete/) | ड्राइंग से रेखा हटाता है |

## गुण

जब एक रेखा चुनी जाती है तो गुण पैनल हर वह फ़ील्ड दिखाता है जो DXF `LINE` रिकॉर्ड में होती है:

**सामान्य**

| गुण | डिफ़ॉल्ट | अर्थ |
|----------|---------|---------|
| Color | 256 (ByLayer) | ACI रंग सूचकांक |
| Layer | `0` | लेयर असाइनमेंट |
| Linetype | ByLayer | नामित लाइनटाइप पैटर्न |
| Linetype Scale | 1 | लाइनटाइप पैटर्न पर स्केल फैक्टर |
| Thickness | 0 | एक्सट्रूज़न मोटाई |

**ज्यामिति**

| गुण | अर्थ |
|----------|---------|
| Start X / Start Y | पहले एंडपॉइंट के निर्देशांक |
| End X / End Y | दूसरे एंडपॉइंट के निर्देशांक |

सभी फ़ील्ड कमांड को फिर से चलाए बिना पैनल में सीधे संपादन योग्य हैं।

## Line बनाम Polyline — कब किसका उपयोग करें

| | Line | Polyline |
|---|------|---------|
| ऑब्जेक्ट गिनती | प्रति खंड एक `LINE` | पूरे पथ के लिए एक `LWPOLYLINE` |
| Trim / Extend | हाँ — खंड दर खंड | नहीं |
| बंद आकार | नहीं | हाँ (close flag) |
| ग्रिप संपादन | व्यक्तिगत एंडपॉइंट स्ट्रेच करें | पथ के साथ कोई भी वर्टेक्स मूव करें |
| सबसे अच्छा | कंस्ट्रक्शन रेखाएं, एकल खंड, जिसे ट्रिम करेंगे | कंटूर, आउटलाइन, पूरे आकार |

## DXF — LINE ऑब्जेक्ट

रेखाएं DXF फ़ाइल में `LINE` ऑब्जेक्ट के रूप में सहेजी जाती हैं। हर गुण — start/end निर्देशांक, रंग, लेयर, लाइनटाइप, लाइनटाइप स्केल, और मोटाई — बिना किसी नुकसान के राउंड-ट्रिप करता है। जब आप `LINE` ऑब्जेक्ट वाली DXF खोलते हैं तो वे एडिटर में पूरी तरह संपादन योग्य `Line` ऑब्जेक्ट बन जाते हैं।

एडिटर में खींची गई रेखाएं सहेजने पर `LINE` ऑब्जेक्ट के रूप में भी लिखी जाती हैं, इसलिए LibreCAD, FreeCAD और किसी भी अन्य DXF-संगत एप्लिकेशन द्वारा पढ़ी जा सकती हैं।
