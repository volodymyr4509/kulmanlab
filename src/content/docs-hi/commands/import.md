# Import

**Import** कमांड आपके लोकल फ़ाइल सिस्टम से मौजूदा ड्राइंग KulmanLab CAD में लोड करती है। मानक **DXF** प्रारूप और KulmanLab का अपना **JSON** प्रारूप दोनों समर्थित हैं।

## फ़ाइल आयात कैसे करें

1. स्क्रीन के शीर्ष पर File पैनल में **Import** टूलबार बटन (फ़ोल्डर आइकन) पर क्लिक करें।
2. आपके ब्राउज़र का फ़ाइल पिकर खुलता है। अपनी ड्राइंग फ़ाइल पर नेविगेट करें और चुनें।
3. ड्राइंग तुरंत कैनवास पर लोड होती है। व्यूपोर्ट स्वचालित रूप से सभी ऑब्जेक्ट फिट करता है।

वैकल्पिक रूप से, फ़ाइल को सीधे कैनवास पर ड्रैग और ड्रॉप कर सकते हैं।

## समर्थित फ़ाइल प्रारूप

| प्रारूप | एक्सटेंशन | कब उपयोग करें |
|--------|----------|--------------|
| **DXF** | `.dxf` | AutoCAD, FreeCAD, LibreCAD, या अन्य CAD टूल्स से ड्राइंग |
| **JSON** *(नेटिव)* | `.json` | KulmanLab CAD से पहले सेव की गई ड्राइंग — पूर्ण फिडेलिटी |

## DXF से क्या आयात होता है

KulmanLab निम्नलिखित DXF ऑब्जेक्ट प्रकार पार्स करता है:

| ऑब्जेक्ट प्रकार | DXF कोड | नोट्स |
|----------------|---------|-------|
| रेखा | `LINE` | |
| वृत्त | `CIRCLE` | |
| चाप | `ARC` | |
| दीर्घवृत्त | `ELLIPSE` | |
| पॉलीलाइन | `LWPOLYLINE` | |
| स्प्लाइन | `SPLINE` | |
| टेक्स्ट | `TEXT`, `MTEXT` | |
| माप | `DIMENSION` | |
| मल्टीलीडर | `MULTILEADER` | |

मौजूद होने पर लेयर परिभाषाएं और लाइनटाइप टेबल भी DXF फ़ाइल से आयात होते हैं।

असमर्थित DXF प्रकार वाले ऑब्जेक्ट चुपचाप छोड़ दिए जाते हैं — बाकी ड्राइंग फिर भी लोड होती है।

## फ़ाइल नामकरण और स्टोरेज

जब आप फ़ाइल आयात करते हैं, तो फ़ाइल नाम में टाइमस्टैंप जोड़ा जाता है (जैसे `myplan_May22_14:30:00.dxf`)। इससे आप [Recent Files](./files) में एक ही ड्राइंग के कई संस्करण नाम टकराव के बिना रख सकते हैं। यदि फ़ाइल नाम में पहले से टाइमस्टैंप है, तो वह वैसे ही उपयोग होता है।

आयात के बाद ड्राइंग स्वचालित रूप से ब्राउज़र स्टोरेज (IndexedDB) में सेव होती है, इसलिए [Files](./files) पैनल में दिखती है और पेज रिलोड होने पर बनी रहती है।

## वर्तमान ड्राइंग का क्या होता है

आयात करने से वर्तमान कैनवास रिप्लेस हो जाता है। कोई मर्ज या अपेंड नहीं। यदि आपके पास असेव किए गए परिवर्तन हैं, तो पहले वर्तमान ड्राइंग [export](./export) करें।

## स्टार्टअप पर

KulmanLab पेज लोड होने पर स्वचालित रूप से सबसे हाल ही में संपादित फ़ाइल फिर से खोलता है। यदि कोई सेव की गई फ़ाइल नहीं है, तो एक डिफ़ॉल्ट सैंपल ड्राइंग लोड होती है।

## समस्या निवारण

| समस्या | संभावित कारण | समाधान |
|--------|------------|--------|
| आयात के बाद कैनवास खाली है | DXF ऑब्जेक्ट असमर्थित प्रकार (जैसे HATCH, INSERT) का उपयोग करते हैं | ऑब्जेक्ट छोड़ दिए गए — टर्मिनल में "no entities found" संदेश देखें |
| Import बटन कुछ नहीं करता | ब्राउज़र ने फ़ाइल पिकर ब्लॉक किया | बटन एक बार और क्लिक करें; कुछ ब्राउज़र को नए यूज़र जेस्चर की आवश्यकता है |
| माप गलत दिखती हैं | ऐसे टूल से DXF जो गैर-मानक माप ज्यामिति लिखता है | करंट DXF संस्करण का उपयोग करके स्रोत ऐप से फिर से निर्यात करें |
