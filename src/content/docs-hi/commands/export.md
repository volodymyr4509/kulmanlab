# Export

**Export** कमांड वर्तमान ड्राइंग आपके फ़ाइल सिस्टम पर डाउनलोड करती है। दो प्रारूप उपलब्ध हैं: अन्य CAD टूल्स के साथ संगतता के लिए **DXF** और KulmanLab CAD के भीतर पूर्ण-फिडेलिटी सेव के लिए **JSON**।

## निर्यात कैसे करें

1. File पैनल में **Export** टूलबार बटन (डाउनलोड आइकन) पर क्लिक करें।
2. **Export Manager** पॉपअप खुलता है।
3. प्रारूप चुनने के लिए फ़ॉर्मेट कार्ड पर क्लिक करें — **JSON** या **DXF**।
4. **Export** बटन पर क्लिक करें। फ़ाइल स्वचालित रूप से आपके डिफ़ॉल्ट डाउनलोड फ़ोल्डर में डाउनलोड होती है।

## प्रारूप चुनना

| प्रारूप | एक्सटेंशन | सबसे अच्छा | सीमाएं |
|--------|----------|----------|--------|
| **JSON** *(नेटिव)* | `.json` | KulmanLab CAD में फिर से खोलने के लिए काम सेव करना | अन्य CAD टूल्स के साथ संगत नहीं |
| **DXF** | `.dxf` | AutoCAD, FreeCAD, LibreCAD आदि के साथ शेयर करना | माप और लीडर निर्यात नहीं होते |

**JSON कब उपयोग करें:** जब भी आप अपने काम की पूरी कॉपी सेव करना चाहते हों। JSON KulmanLab का नेटिव प्रारूप है और हर ऑब्जेक्ट को बिल्कुल सुरक्षित रखता है — माप, लीडर, और सभी लेयर डेटा सहित।

**DXF कब उपयोग करें:** जब आपको किसी अन्य CAD एप्लिकेशन का उपयोग करने वाले व्यक्ति को ड्राइंग देनी हो। निर्यात की गई फ़ाइल AC1012 DXF प्रारूप का उपयोग करती है और अधिकांश DXF-संगत टूल्स में खोली जा सकती है।

## प्रारूप के अनुसार क्या निर्यात होता है

### JSON निर्यात

सभी ऑब्जेक्ट प्रकार शामिल हैं:

- रेखाएं, वृत्त, चाप, दीर्घवृत्त, पॉलीलाइन, स्प्लाइन, टेक्स्ट
- माप (रैखिक, संरेखित, जारी, त्रिज्या, व्यास)
- मल्टीलीडर
- लेयर परिभाषाएं और लाइनटाइप टेबल

### DXF निर्यात

केवल ज्यामिति ऑब्जेक्ट शामिल हैं:

- रेखाएं, वृत्त, चाप, दीर्घवृत्त, पॉलीलाइन (`LWPOLYLINE` के रूप में निर्यात), स्प्लाइन, टेक्स्ट
- लेयर परिभाषाएं और लाइनटाइप टेबल

**DXF में शामिल नहीं:** माप ऑब्जेक्ट और मल्टीलीडर। ये KulmanLab-विशिष्ट डेटा संरचनाओं का उपयोग करते हैं जिन्हें मानक DXF में विश्वसनीय रूप से प्रस्तुत नहीं किया जा सकता। यदि आपकी ड्राइंग में एनोटेशन हैं, तो JSON का उपयोग करें या उन्हें विज़ुअली कैप्चर करने के लिए [Print](../print/) का उपयोग करें।

## निर्यात की गई फ़ाइल का नाम

डाउनलोड की गई फ़ाइल वर्तमान ड्राइंग फ़ाइल के नाम पर रखी जाती है (जैसे `myplan_May22_14:30:00.json`)। एक्सटेंशन चुने गए प्रारूप से मेल खाने के लिए बदल जाता है।

## Export और Print का अंतर

| फ़ीचर | Export | Print |
|-------|--------|-------|
| आउटपुट | वेक्टर सोर्स फ़ाइल (.dxf / .json) | रास्टर इमेज (.png / .jpeg / .webp / .pdf) |
| अन्य टूल्स में संपादन योग्य | हाँ (DXF) | नहीं |
| लेयर और लाइनटाइप सुरक्षित | हाँ | नहीं (फ्लैट रेंडर होता है) |
| माप और लीडर कैप्चर | केवल JSON | हाँ |

**Export** का उपयोग तब करें जब संपादन योग्य फ़ाइल की आवश्यकता हो। [Print](../print/) का उपयोग तब करें जब विज़ुअल स्नैपशॉट की आवश्यकता हो।
