# Leader

`leader` कमांड चार चरणों में एक मल्टीलीडर एनोटेशन बनाती है: एक तीर जो एक फीचर को छूता है, एक लीडर रेखा जो डॉगलेग पर झुकती है, एक टेक्स्ट एंकर, और एक टाइप किया गया लेबल। सभी एनोटेशन कमांड में से, Leader एकमात्र ऐसी है जिसमें ब्लिंकिंग कर्सर प्रीव्यू के साथ इंटरैक्टिव टेक्स्ट-इनपुट चरण शामिल है।

## मल्टीलीडर की संरचना

```
  ◄── arrowhead tip  (step 2 — touches the feature)
      \
       \  leader line
        \
         ●──── dogleg (step 3) ──── text anchor (step 4)
                                    Label text  (step 5)
```

- **तीर टिप** — एनोटेट किए जाने वाले फीचर पर रखा गया नुकीला सिरा।
- **डॉगलेग** — वह कोहनी जहाँ लीडर रेखा टेक्स्ट की ओर झुकती है।
- **टेक्स्ट एंकर** — जहाँ लेबल स्थित है। टेक्स्ट स्वचालित रूप से बाएं या दाएं संरेखित होता है।

## लीडर बनाना

1. टर्मिनल में `leader` टाइप करें या **Leader** टूलबार बटन पर क्लिक करें।
2. **तीर टिप पर क्लिक करें**, या सटीक निर्देशांक के लिए `X,Y` टाइप करें और **Enter** दबाएं।
3. **डॉगलेग पर क्लिक करें** — लीडर में मोड़। कोण 45° वृद्धि पर लॉक करता है; सटीक प्लेसमेंट के लिए लंबाई टाइप करें और **Enter** दबाएं। या `X,Y` टाइप करें निरपेक्ष निर्देशांक दर्ज करने के लिए।
4. **टेक्स्ट स्थिति पर क्लिक करें** — जहाँ लेबल एंकर होगा। समान विकल्प लागू होते हैं: क्लिक, कोण-लॉक + लंबाई, या `X,Y`।
5. **लेबल टेक्स्ट टाइप करें** — कैनवास प्रीव्यू ब्लिंकिंग कर्सर के साथ लाइव अपडेट होता है। रखने के लिए **Enter** दबाएं।

## निर्देशांक प्रविष्टि (सभी बिंदु चरण)

किसी भी बिंदु-चुनने के चरण (टिप, डॉगलेग, टेक्स्ट स्थिति) पर क्लिक करने के बजाय सटीक निर्देशांक टाइप कर सकते हैं:

1. X मान टाइप करें (अंक, `.`, या `-`)।
2. `,` दबाएं — टर्मिनल `[X], [Y{cursor}]` दिखाता है कि X लॉक है।
3. Y मान टाइप करें।
4. बिंदु रखने के लिए **Enter** दबाएं।

## कोण लॉकिंग (चरण 3 और 4)

प्रत्येक रखे गए बिंदु के बाद, कमांड 45° अक्षों पर स्नैप करती है जब कर्सर काफी दूर हो। लॉक होने पर:
- प्रीव्यू अक्ष पर स्नैप हो जाता है।
- एक लंबाई टाइप करें और अगला बिंदु उस सटीक दूरी पर रखने के लिए **Enter** दबाएं।

कोण लॉकिंग और निर्देशांक प्रविष्टि परस्पर अनन्य हैं — एक बार जब आप पूर्व `,` के बिना एक अंक टाइप करते हैं, कमांड इसे दूरी के रूप में व्याख्यायित करती है (कोण लॉक सक्रिय होना चाहिए)। इसके बजाय निरपेक्ष निर्देशांक दर्ज करने के लिए, X संख्या से शुरू करें और कॉमा का पालन करें।

## टेक्स्ट लेबल संपादन

चरण 5 में लेबल टाइप करते समय, रखने से पहले टेक्स्ट नेविगेट और संपादित कर सकते हैं:

| Key | Action |
|-----|--------|
| कोई भी प्रिंट करने योग्य अक्षर | कर्सर स्थिति पर डालें |
| `←` / `→` | कर्सर बाएं या दाएं मूव करें |
| `Backspace` | कर्सर के बाईं ओर का अक्षर हटाएं |
| `Delete` | कर्सर के दाईं ओर का अक्षर हटाएं |
| `Enter` | लीडर रखें |

## स्वचालित टेक्स्ट दिशा

टेक्स्ट संरेखण डॉगलेग के सापेक्ष कर्सर स्थिति के अनुसार समायोजित होता है:

| कर्सर स्थिति | टेक्स्ट दिशा |
|-----------------|---------------|
| डॉगलेग के **दाईं ओर** | टेक्स्ट एंकर से बाएं-से-दाएं |
| डॉगलेग के **बाईं ओर** | दाएं-से-बाएं (दाईं ओर एंकर) |

कोई मैन्युअल समायोजन आवश्यक नहीं — कर्सर को उस साइड पर ले जाएं जहाँ आप लेबल चाहते हैं और यह सही ढंग से संरेखित होता है।

## कीबोर्ड संदर्भ

**बिंदु चरण (टिप, डॉगलेग, टेक्स्ट स्थिति)**

| Key | Action |
|-----|--------|
| `0`–`9`, `.`, `-` | X निर्देशांक टाइप करना शुरू करें (फिर X लॉक करने और Y दर्ज करने के लिए `,`) |
| `,` | X की पुष्टि करें और Y प्रविष्टि पर जाएं |
| `0`–`9`, `.`, `-` | कोण-लॉक होने पर दूरी बनाएं |
| `Backspace` | अंतिम टाइप किया गया अक्षर हटाएं |
| `Enter` | टाइप किए गए निर्देशांक या दूरी की पुष्टि करें |

**टेक्स्ट इनपुट चरण**

| Key | Action |
|-----|--------|
| प्रिंट करने योग्य अक्षर | कर्सर पर डालें |
| `←` / `→` | कर्सर मूव करें |
| `Backspace` | बाईं ओर हटाएं |
| `Delete` | दाईं ओर हटाएं |
| `Enter` | लीडर रखें |

| Key | Action |
|-----|--------|
| `Escape` | रद्द करें और चरण 2 पर रीसेट करें |

## मौजूदा लीडर संपादित करना

रखे गए multileader पर **डबल-क्लिक** करें **रिच** मोड में टेक्स्ट एडिटर फिर से खोलने के लिए। रिच मोड में आप बोल्ड, इटैलिक, और प्रति-अक्षर फ़ॉन्ट या हाइट ओवरराइड लागू कर सकते हैं, और `Enter` से लाइन ब्रेक डाल सकते हैं। कमिट और बंद करने के लिए **Escape** दबाएं।

पूर्ण संदर्भ के लिए [Text Editor — rich mode](../../interface/text-editor/#rich-mode) देखें।

## भुजाएं जोड़ना और हटाना

- मौजूदा लीडर में अतिरिक्त तीर भुजा जोड़ने के लिए: [Leader+](../leader-add/)
- दो या अधिक भुजाओं वाले लीडर से भुजा हटाने के लिए: [Leader−](../leader-remove/)

## DXF — केवल इम्पोर्ट

**लीडर केवल इम्पोर्ट के लिए हैं।** DXF फ़ाइलों से `MLEADER` ऑब्जेक्ट सही तरह से पढ़े और प्रदर्शित किए जाते हैं, लेकिन एडिटर में बनाए गए लीडर DXF फ़ाइल सहेजने पर **नहीं लिखे** जाते। दृश्य एनोटेशन के लिए लीडर का उपयोग करें; राउंड-ट्रिप वर्कफ़्लो के लिए उन पर निर्भर न रहें।
