# Import

คำสั่ง **Import** โหลดภาพวาดที่มีอยู่จากระบบไฟล์ในเครื่องของคุณไปยัง KulmanLab CAD รองรับทั้งรูปแบบ **DXF** มาตรฐานและรูปแบบ **JSON** ของ KulmanLab เอง

## วิธีนำเข้าไฟล์

1. คลิกปุ่ม **Import** บนแถบเครื่องมือ (ไอคอนโฟลเดอร์) ในแผง File ที่ด้านบนของหน้าจอ
2. ตัวเลือกไฟล์ของเบราว์เซอร์เปิด ไปยังไฟล์ภาพวาดของคุณและเลือก
3. ภาพวาดโหลดบนแคนวาสทันที พื้นที่แสดงผลจัดวัตถุทั้งหมดโดยอัตโนมัติ

อีกทางหนึ่ง คุณสามารถลากและวางไฟล์โดยตรงบนแคนวาส

## รูปแบบไฟล์ที่รองรับ

| รูปแบบ | นามสกุล | เมื่อใช้ |
|--------|---------|---------|
| **DXF** | `.dxf` | ภาพวาดจาก FreeCAD, LibreCAD หรือเครื่องมือ CAD อื่นๆ |
| **JSON** *(native)* | `.json` | ภาพวาดที่บันทึกก่อนหน้าจาก KulmanLab CAD — ความเที่ยงตรงเต็มรูปแบบ |

## สิ่งที่นำเข้าจาก DXF

KulmanLab แยกวิเคราะห์ประเภทวัตถุ DXF ต่อไปนี้:

| ประเภทวัตถุ | รหัส DXF | หมายเหตุ |
|------------|---------|----------|
| Line | `LINE` | |
| Circle | `CIRCLE` | |
| Arc | `ARC` | |
| Ellipse | `ELLIPSE` | |
| Polyline | `LWPOLYLINE` | |
| Spline | `SPLINE` | |
| Text | `TEXT`, `MTEXT` | |
| Dimension | `DIMENSION` | |
| Multileader | `MULTILEADER` | |

คำจำกัดความเลเยอร์และตาราง linetype ยังถูกนำเข้าจากไฟล์ DXF เมื่อมี

วัตถุที่ใช้ประเภท DXF ที่ไม่รองรับถูกข้ามอย่างเงียบๆ — ส่วนที่เหลือของภาพวาดยังโหลด

## การตั้งชื่อไฟล์และการจัดเก็บ

เมื่อคุณนำเข้าไฟล์ การประทับเวลาถูกเพิ่มต่อท้ายชื่อไฟล์ (เช่น `myplan_May22_14:30:00.dxf`) ซึ่งช่วยให้คุณเก็บหลายเวอร์ชันของภาพวาดเดิมใน [Recent Files](../files/) โดยไม่มีการชนกันของชื่อ หากชื่อไฟล์มีการประทับเวลาแล้ว จะใช้ตามที่เป็น

ภาพวาดถูกบันทึกไปยังหน่วยเก็บข้อมูลเบราว์เซอร์ (IndexedDB) โดยอัตโนมัติหลังจากนำเข้า ดังนั้นจึงปรากฏในแผง [Files](../files/) และรอดชีวิตจากการโหลดหน้า

## สิ่งที่เกิดขึ้นกับภาพวาดปัจจุบัน

การนำเข้าแทนที่แคนวาสปัจจุบัน ไม่มีการรวมหรือเพิ่มต่อท้าย หากมีการเปลี่ยนแปลงที่ยังไม่ได้บันทึก [ส่งออก](../export/) ภาพวาดปัจจุบันก่อน

## เมื่อเริ่มต้น

KulmanLab จะเปิดไฟล์ที่แก้ไขล่าสุดโดยอัตโนมัติเมื่อโหลดหน้า หากไม่มีไฟล์ที่บันทึก ภาพวาดตัวอย่างเริ่มต้นจะถูกโหลด

## การแก้ปัญหา

| ปัญหา | สาเหตุที่เป็นไปได้ | การแก้ไข |
|-------|----------------|----------|
| แคนวาสว่างหลังนำเข้า | วัตถุ DXF ใช้ประเภทที่ไม่รองรับ (เช่น HATCH, INSERT) | วัตถุถูกข้าม — ตรวจสอบข้อความ "no entities found" ในเทอร์มินัล |
| ปุ่มนำเข้าไม่ทำงาน | เบราว์เซอร์บล็อกตัวเลือกไฟล์ | คลิกปุ่มอีกครั้ง บางเบราว์เซอร์ต้องการท่าทางผู้ใช้ใหม่ |
| มิติดูผิดปกติ | DXF จากเครื่องมือที่เขียนเรขาคณิตมิติไม่ตรงมาตรฐาน | ส่งออกใหม่จากแอปต้นทางโดยใช้เวอร์ชัน DXF ปัจจุบัน |

## คำสั่งที่เกี่ยวข้อง

- [Files](../files/) — เรียกดูและกู้คืนภาพวาดที่บันทึกในเบราว์เซอร์
- [New File](../new-file/) — เริ่มภาพวาดเปล่า
