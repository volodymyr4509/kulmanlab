# Vector Pins

**Vector Pins** คือเครื่องมือช่วยวาดที่ให้คุณจัดแนวรูปทรงใหม่กับจุดที่มีอยู่โดยไม่ต้องวาดเส้นช่วยร่าง ชี้เคอร์เซอร์ค้างบนจุดสแนปครึ่งวินาทีเพื่อ*ปักหมุด*จุดนั้น — หมุดจะฉายเส้นอ้างอิงแนวนอนและแนวตั้งที่มองไม่เห็น และเคอร์เซอร์จะสแนปเข้ากับเส้นเหล่านั้นทุกครั้งที่เข้าใกล้ นี่คือฟีเจอร์เทียบเท่าการติดตามสแนปวัตถุ (object snap tracking) ของแอป CAD บนเดสก์ท็อปใน KulmanLab CAD

ฟีเจอร์นี้ควบคุมด้วยปุ่มสลับ **Pins** ในแถบควบคุม (ข้าง Grid, Snap และ ANGL) โดย**เปิดใช้เป็นค่าเริ่มต้น** และการตั้งค่าจะคงอยู่ระหว่างเซสชัน

## การปักหมุดจุด

1. เริ่มคำสั่งที่ต้องการจุด — [Line](../../commands/line/), [Circle](../../commands/circle/), [Move](../../commands/move/) เป็นต้น
2. เลื่อนเคอร์เซอร์ไปบนจุดสแนปของรูปทรงที่มีอยู่ — เครื่องหมายจุดปลาย จุดกึ่งกลาง หรือจุดศูนย์กลาง
3. **ค้างเคอร์เซอร์นิ่งไว้ 500 มิลลิวินาที** เครื่องหมายจะเปลี่ยนเป็น**สี่เหลี่ยม**ทึบสีเน้น — จุดถูกปักหมุดแล้ว
4. ทำซ้ำเพื่อปักหมุดกี่จุดก็ได้ตามต้องการ แต่ละหมุดจะฉายเส้นอ้างอิงของมันต่อไป

การปักหมุดยังทำงานนอกคำสั่งด้วย: การชี้เคอร์เซอร์บน**กริป**ของวัตถุที่เลือกอยู่จะปักหมุดมันในแบบเดียวกัน

## การติดตามตามเส้นอ้างอิง

จุดที่ปักหมุดแต่ละจุดจะฉายเส้นอ้างอิงที่มองไม่เห็นสองเส้น — **แนวนอน**หนึ่งเส้นและ**แนวตั้ง**หนึ่งเส้น — ผ่านพิกัดที่แน่นอนของมัน เมื่อคุณเลื่อนเคอร์เซอร์:

- ภายใน **12 px** จากเส้นแนวตั้งของหมุด เคอร์เซอร์จะสแนปเข้ากับเส้นนั้น: เส้นประสีเน้นถูกวาดผ่านหมุดตลอดทั้งมุมมอง และ**เครื่องหมาย X** แสดงตำแหน่งที่สแนป พิกัด X ของคุณตอนนี้เท่ากับ X ของหมุด*พอดี*
- เช่นเดียวกันกับเส้นแนวนอนสำหรับพิกัด Y ของหมุด
- เมื่ออยู่ใกล้เส้นของแต่ละแนวอย่างละหนึ่งเส้น — แม้มาจาก**หมุดสองตัวที่ต่างกัน** — เคอร์เซอร์จะสแนปเข้ากับ**จุดตัด**ของเส้นทั้งสอง และเส้นประทั้งคู่จะแสดงขึ้น ทำให้วางจุดได้ตรง (X ของหมุด A, Y ของหมุด B) พอดี

```
                    ┆ (เส้นประ เส้นแนวตั้งของหมุด ■)
                    ┆
   ■ หมุด A ┄┄┄┄┄┄┄ ✕ ← เคอร์เซอร์สแนปที่จุดตัด:
                    ┆    X จากหมุด B, Y จากหมุด A
                    ┆
                    ■ หมุด B
```

พิกัดที่สแนปถูกดึงมาจากหมุดโดยตรง การจัดแนวจึงแม่นยำ — ไม่มีการปัดเศษหรือความคลาดเคลื่อนจากเลขทศนิยม

## ลำดับความสำคัญของการสแนป

การสแนปรูปทรงปกติ — จุดปลาย จุดกึ่งกลาง จุดศูนย์กลาง และจุดตัด — **มีความสำคัญเหนือกว่า**เส้นอ้างอิงของหมุด หากเคอร์เซอร์อยู่ใกล้จุดสแนปมากกว่าเส้นอ้างอิง จุดสแนปจะชนะ การติดตามหมุดเติมช่องว่างระหว่างรูปทรง ไม่เคยขวางการสแนปเข้ากับรูปทรงเอง

## การใช้ร่วมกับการล็อกมุม

Vector pins ทำงานร่วมกับการติดตามมุม (ปุ่มสลับ **ANGL** ในแถบควบคุม) เมื่อคำสั่งล็อกเคอร์เซอร์ไว้บนรังสีติดตามมุม:

- เคอร์เซอร์ยังคงถูกจำกัดอยู่ในทิศทางที่ล็อก
- การสแนปหมุดจะเปลี่ยนไปเล็ง**จุดตัดของรังสีที่ล็อกกับเส้นอ้างอิงของหมุด** (เฉพาะด้านหน้าจุดกำเนิดของรังสีเท่านั้น)

สิ่งนี้ตอบคำถามอย่าง *"ทิศทาง 45° จากจุดสุดท้ายของฉันตัดกับระดับความสูงของจุดศูนย์กลางวงกลมนั้นตรงไหน?"* — ล็อกมุม แล้วเคอร์เซอร์จะเข้าล็อกที่จุดตัดพอดี การสแนปตามรังสีทำงานในทุกคำสั่งที่มีการล็อกมุม: Line, Polyline, Arc, Circle, Move, Copy, Area, Leader และ ViewportCopy

## วงจรชีวิตของหมุด

หมุดมีไว้สำหรับการทำงานที่กำลังดำเนินอยู่ ไม่ใช่เครื่องหมายถาวร หมุดทั้งหมดจะถูกล้างเมื่อ:

| เหตุการณ์ | เหตุผล |
|-----------|--------|
| **คำสั่งใหม่**เริ่มทำงาน | ทุกการทำงานเริ่มด้วยชุดอ้างอิงที่สะอาด |
| กด **Escape** | พฤติกรรมยกเลิกทุกอย่างตามมาตรฐาน |
| ปิดปุ่มสลับ **Pins** | การปิดฟีเจอร์จะลบสถานะของมัน |
| สลับระหว่าง**พื้นที่โมเดลกับพื้นที่กระดาษ** | พิกัดหมุดผูกกับพื้นที่เดียวเท่านั้น |

ภายในคำสั่งเดียว คุณสามารถปักหมุด วาด ปักหมุดอีก แล้วทำต่อได้ — หมุดคงอยู่ผ่านทุกคลิกของคำสั่งหลายจุดอย่าง Polyline

## ขั้นตอนการทำงานทั่วไป

วาดเส้นตรงที่เริ่มต้นตรงใต้จุดศูนย์กลางของวงกลมพอดี:

1. พิมพ์ `line` (หรือคลิกปุ่ม Line)
2. ชี้เคอร์เซอร์ค้างบน**เครื่องหมายจุดศูนย์กลาง**ของวงกลมครึ่งวินาที — มันจะกลายเป็นสี่เหลี่ยมสีเน้น
3. เลื่อนเคอร์เซอร์ลงด้านล่าง: เมื่อใกล้แนวตั้งของวงกลม เคอร์เซอร์จะล็อกเข้ากับเส้นอ้างอิงแบบเส้นประ
4. คลิก — เส้นเริ่มต้นตรงพิกัด X ของวงกลมพอดี
5. วาดเส้นต่อตามปกติ หมุดยังคงพร้อมใช้สำหรับจุดถัดไป

## หมายเหตุ

- การค้าง 500 มิลลิวินาทีทำงานกับเครื่องหมายสแนปใดก็ตามที่เคอร์เซอร์เข้าถึงได้ — รวมถึงจุดสแนปที่ปรากฏระหว่างคำสั่ง
- การชี้ค้างบนจุดที่ปักหมุดแล้วจะไม่มีผลใด ๆ ไม่มีการถอนหมุดด้วยการชี้ค้าง ล้างหมุดด้วย **Escape** หรือปิด **Pins**
- ระยะสแนปของเส้นอ้างอิงคือ 12 พิกเซลหน้าจอเท่ากับการสแนปจุดปกติ ความรู้สึกจึงสม่ำเสมอในทุกระดับซูม
- จุดที่ปักหมุดแสดงเป็นสี่เหลี่ยมสีเน้นแทนเครื่องหมายสแนปปกติ
