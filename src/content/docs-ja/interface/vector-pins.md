# Vector Pins

**Vector Pins**（ベクターピン）は、補助線を引かずに新しい図形を既存の点に整列させる作図支援機能です。スナップ点に半秒カーソルを合わせると*ピン留め*されます — ピンは目に見えない水平・垂直の参照線を投影し、カーソルが近づくとその線にスナップします。デスクトップ CAD アプリのオブジェクトスナップトラッキングに相当する KulmanLab CAD の機能です。

この機能はコントロールバーの **Pins** トグル（Grid・Snap・ANGL の隣）で制御します。**デフォルトで有効**で、設定はセッションをまたいで保持されます。

## 点をピン留めする

1. 点の入力を求めるコマンドを開始します — [Line](../../commands/line/)、[Circle](../../commands/circle/)、[Move](../../commands/move/) など。
2. 既存図形のスナップ点 — 端点・中点・中心のマーカー — にカーソルを合わせます。
3. **カーソルを 500 ms 静止させます。** マーカーが塗りつぶしのアクセント色の**四角形**に変わります — 点がピン留めされました。
4. 必要なだけ繰り返してピン留めできます。各ピンは参照線を投影し続けます。

ピン留めはコマンド外でも機能します。選択中エンティティの**グリップ**にカーソルを合わせると、同じようにピン留めされます。

## 参照線に沿った追跡

ピン留めされた各点は、その正確な座標を通る 2 本の見えない参照線 — **水平**と**垂直** — を投影します。カーソルを動かすと：

- ピンの垂直線から **12 px** 以内でカーソルがその線にスナップします。ピンを通る破線のアクセント色の線がビュー全体に描かれ、**X マーカー**がスナップ位置を示します。X 座標はピンの X と*完全に一致*します。
- ピンの Y 座標に対する水平線も同様です。
- 各方向の線 1 本ずつの近く — **異なる 2 つのピン**からでも — カーソルはそれらの**交点**にスナップし、両方の破線が表示されます。これで点を正確に（ピン A の X、ピン B の Y）に配置できます。

```
                    ┆ （破線、ピン ■ の垂直線）
                    ┆
   ■ ピン A ┄┄┄┄┄┄┄ ✕ ← カーソルは交点にスナップ：
                    ┆    X はピン B から、Y はピン A から
                    ┆
                    ■ ピン B
```

スナップされた座標はピンから直接取得されるため、整列は正確です — 丸めや浮動小数点の誤差はありません。

## スナップの優先順位

通常のジオメトリスナップ — 端点・中点・中心・交点 — はピンの参照線より**優先**されます。カーソルが参照線より点スナップに近い場合は、点スナップが勝ちます。ピン追跡は図形の間の空白を埋めるものであり、図形そのものへのスナップを妨げることはありません。

## 角度ロックとの組み合わせ

Vector pins は角度トラッキング（コントロールバーの **ANGL** トグル）と連携します。コマンドがカーソルを角度トラッキングのレイにロックしている場合：

- カーソルはロックされた方向に拘束されたままです。
- ピンスナップは**ロックされたレイとピン参照線の交点**を対象に切り替わります（レイの原点より前方のみ）。

これは*「最後の点からの 45° 方向は、あの円の中心の高さとどこで交わるか？」*のような疑問に答えます — 角度をロックすれば、カーソルが交差点にカチッと収まります。レイスナップは角度ロックを持つすべてのコマンドで動作します：Line、Polyline、Arc、Circle、Move、Copy、Area、Leader、ViewportCopy。

## ピンのライフサイクル

ピンは目前の操作のためのもので、永続的なマーカーではありません。次の場合にすべてのピンがクリアされます：

| イベント | 理由 |
|---------|------|
| **新しいコマンド**が起動 | 各操作はクリーンな参照セットで開始 |
| **Escape** を押す | 標準の「すべてキャンセル」動作 |
| **Pins** トグルをオフにする | 機能をオフにすると状態も消去 |
| **モデル空間とペーパー空間**の切り替え | ピン座標は一方の空間に固有 |

1 つのコマンド内では、ピン留め → 作図 → 再ピン留めと続けられます — Polyline のような複数点コマンドの各クリック後もピンは残ります。

## 典型的なワークフロー

円の中心の真下から始まる直線を引く：

1. `line` と入力します（または Line ボタンをクリック）。
2. 円の**中心マーカー**に半秒カーソルを合わせます — アクセント色の四角形に変わります。
3. カーソルを下へ動かします。円の垂直線の近くで、カーソルが破線の参照線にロックされます。
4. クリックします — 直線は円の X 座標ちょうどから始まります。
5. あとは通常どおり直線を続けます。ピンは次の点でも利用できます。

## 補足

- 500 ms のホバーは、カーソルが届くあらゆるスナップマーカーで機能します — コマンドの途中で現れるスナップ点も含みます。
- すでにピン留めされた点にカーソルを合わせても何も起こりません。ホバーによるピン解除はありません。ピンのクリアは **Escape** か **Pins** のオフで行います。
- 参照線のスナップ距離は通常の点スナップと同じ画面上 12 ピクセルなので、どのズームレベルでも操作感は一貫しています。
- ピン留めされた点は、通常のスナップマーカーの代わりにアクセント色の四角形で描画されます。
