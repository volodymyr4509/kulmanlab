# Redo（やり直し）

`redo` コマンドは[Undo](../undo/)で元に戻した操作を再適用して、取り消し履歴を前進させます。Redo は Undo でステップを戻した後、新しい変更を加えていない場合にのみ使用できます。

## やり直しの方法

- ターミナルに `redo` と入力する、または
- **Redo** ツールバーボタンをクリックします。

各実行で以前に取り消した操作を 1 つ再適用します。繰り返し実行すると、利用可能なすべての Redo エントリを前進させることができます。

## Redo スタックの動作

| 詳細 | 動作 |
|-----|------|
| 利用可能になる条件 | 1 回以上の [Undo](../undo/) ステップ後 |
| クリアされる条件 | **新しい作図操作** — エンティティの追加、編集、または削除 |
| ストレージ | ブラウザー、ファイルごと — ページリロード後も保持（リロード前に新しい操作が行われていない場合） |
| 最大深度 | 最大 20 エントリ（Undo と同じプール） |

新しいエンティティが作図、削除、または変更されると、Redo スタックはクリアされてそれらのエントリは回復できません。新しい作業に置き換えられていない取り消し済みの操作のみやり直せます。

## Redo と Undo の比較

| | Redo | Undo |
|---|------|------|
| 方向 | 取り消し済みエントリを**前進** | 履歴を**後退** |
| 利用可能な条件 | Undo を少なくとも 1 回行い、新しい操作がない | 記録済みの操作が少なくとも 1 つある |
| クリアされる条件 | 新しい作図操作 | なし |

Redo できるエントリがない場合、ツールバーの Redo ボタンはグレーアウトされます。まず [Undo](../undo/) を使用して Redo エントリを作成してください。
