# Dimension Aligned

`dimaligned` کمانڈ ایک ایسی پیمائش رکھتی ہے جو دو نقاط کے درمیان **اصل سیدھی لائن کا فاصلہ** ناپتی ہے۔ پیمائش لائن دونوں نقاط کو جوڑنے والی لائن کے متوازی چلتی ہے، لہذا یہ کسی بھی زاویے پر ہو سکتی ہے۔ یہ [Dimension Linear](../dim-linear/) سے اہم فرق ہے، جو افقی یا عمودی تک محدود ہے۔

## سیدھ شدہ پیمائش کا خاکہ

```
     ●  p2
    /|
   / |  (ext line 2, پیمائش لائن سے عمودی)
  /  |
 /←5.00→/
/  /
●  /  (ext line 1, پیمائش لائن سے عمودی)
p1
```

- **ایکسٹینشن لائنیں** — پیمائش لائن سے عمودی، ہر ناپے گئے نقطے سے کھینچی گئیں۔
- **پیمائش لائن** — p1→p2 کے متوازی، کرسر کی پوزیشن سے ایک طرف آفسیٹ۔
- **قدر** — اصل یوکلیڈین فاصلہ `|p1 – p2|`۔

## سیدھ شدہ پیمائش رکھنا

1. ٹرمینل میں `dimaligned` ٹائپ کریں یا **Dimension Aligned** ٹول بار بٹن پر کلک کریں۔
2. **پہلی ایکسٹینشن لائن کی اصل پر کلک کریں** (p1)، یا درست کوآرڈینیٹ کے لیے `X,Y` ٹائپ کریں اور **Enter** دبائیں۔
3. **دوسری ایکسٹینشن لائن کی اصل پر کلک کریں** (p2)۔ کوآرڈینیٹ اندراج یہاں بھی کام کرتا ہے۔
4. **کرسر ایک طرف لے جائیں** پیمائش لائن کا عمودی آفسیٹ سیٹ کرنے کے لیے۔
5. **کلک کریں** رکھنے کے لیے، یا آفسیٹ فاصلہ ٹائپ کریں اور درست رکھاؤ کے لیے **Enter** دبائیں۔

## ٹائپ کردہ آفسیٹ فاصلہ

رکھاؤ کے دوران نمبر ٹائپ کریں پیمائش لائن کو p1→p2 لائن سے بالکل عمودی فاصلے پر ٹھیک کرنے کے لیے:

| کلید | عمل |
|-----|--------|
| `0`–`9`, `.` | آفسیٹ میں ہندسہ شامل کریں |
| `Backspace` | آخری ٹائپ کیا گیا حرف حذف کریں |
| `Enter` / `Space` | ٹائپ کیے گئے آفسیٹ پر رکھیں |

کرسر کی طرف طے کرتی ہے کہ پیمائش لائن کس طرف ظاہر ہوگی۔

## کی بورڈ حوالہ

| کلید | عمل |
|-----|--------|
| `0`–`9`, `.`, `-` | X کوآرڈینیٹ اندراج شروع کریں (p1/p2 مراحل)، یا آفسیٹ فاصلہ (رکھاؤ مرحلہ) |
| `,` | X لاک کریں اور Y اندراج پر جائیں (p1/p2 مراحل) |
| `Backspace` | آخری ٹائپ کیا گیا حرف حذف کریں |
| `Enter` / `Space` | ٹائپ کیا گیا کوآرڈینیٹ یا آفسیٹ تصدیق کریں |
| `Escape` | منسوخ کریں |

## Dimension Aligned بمقابلہ Dimension Linear

| | Dimension Aligned | Dimension Linear |
|---|------------------|-----------------|
| پیمائش لائن زاویہ | p1→p2 کے متوازی — کوئی بھی زاویہ | ہمیشہ افقی یا عمودی |
| ناپتا ہے | اصل یوکلیڈین فاصلہ | صرف X یا Y جز |
| H/V رخ لاک | نہیں | ہاں — `H` اور `V` کلیدیں |
| بہترین | ترچھی خصوصیات، زاویہ دار کٹ | ہم مثلث لے آؤٹ، گرڈ سیدھ حصے |

## لیبل ترمیم کرنا — سادہ موڈ

رکھی گئی سیدھ شدہ پیمائش پر **ڈبل کلک** کریں **سادہ** موڈ میں متن ایڈیٹر کھولنے کے لیے۔ ایڈیٹر میں موجودہ رینڈر شدہ قدر پہلے سے بھری ہوتی ہے تاکہ آپ کرسر پوزیشن کر کے براہ راست ترمیم کر سکیں۔

| خصوصیت | سلوک |
|---------|-----------|
| Bold / Italic / Font / Height | **پورے** لیبل پر یکبارگی لاگو ہوتی ہے |
| فی حرف فارمیٹنگ | حمایت یافتہ نہیں |
| `Enter` | قدر محفوظ کرتا ہے اور ایڈیٹر بند کرتا ہے |
| ملٹی لائن | حمایت یافتہ نہیں |

مکمل حوالہ کے لیے [Text Editor — simple mode](../../interface/text-editor/#simple-mode) دیکھیں۔

## پیمائشیں جوڑنا

اس کی دوسری ایکسٹینشن لائن سے جاری رہنے والی مزید پیمائشیں شامل کرنے کے لیے، [Dimension Continue](../dim-continue/) استعمال کریں — یہ اس سیدھ شدہ پیمائش کے ایک ہی پیمائش زاویے پر لاک ہوتی ہے۔

## DXF — DIMENSION شے (سیدھ شدہ قسم)

سیدھ شدہ پیمائشیں DXF میں `dimType = 1` (aligned) کے ساتھ `DIMENSION` اشیاء کے طور پر محفوظ ہوتی ہیں۔ ایکسٹینشن لائن اصل، پیمائش لائن پوزیشن، متن پوزیشن، ناپی گئی قدر، گھماؤ، تیر کا انداز، اور تمام ڈسپلے فلیگز بغیر نقصان کے راؤنڈ ٹرپ کرتے ہیں۔
