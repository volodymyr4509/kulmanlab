# Import

**Import** کمانڈ آپ کے مقامی فائل سسٹم سے موجودہ ڈرائنگ KulmanLab CAD میں لوڈ کرتی ہے۔ معیاری **DXF** فارمیٹ اور KulmanLab کا اپنا **JSON** فارمیٹ دونوں سپورٹ ہیں۔

## فائل درآمد کرنے کا طریقہ

1. اسکرین کے اوپر File پینل میں **Import** ٹول بار بٹن (فولڈر آئیکن) پر کلک کریں۔
2. آپ کے براؤزر کا فائل پِکر کھلتا ہے۔ اپنی ڈرائنگ فائل تک جائیں اور اسے منتخب کریں۔
3. ڈرائنگ فوری طور پر کینوس پر لوڈ ہوتی ہے۔ ویوپورٹ تمام اشیاء کے مطابق خود بخود فٹ ہو جاتا ہے۔

متبادل کے طور پر، آپ فائل کو براہ راست کینوس پر ڈریگ اینڈ ڈراپ کر سکتے ہیں۔

## سپورٹ شدہ فائل فارمیٹس

| فارمیٹ | ایکسٹینشن | کب استعمال کریں |
|--------|-----------|-------------|
| **DXF** | `.dxf` | AutoCAD، FreeCAD، LibreCAD، یا دیگر CAD ٹولز سے ڈرائنگیں |
| **JSON** *(مقامی)* | `.json` | KulmanLab CAD سے پہلے محفوظ کی گئی ڈرائنگیں — مکمل معیار |

## DXF سے کیا درآمد ہوتا ہے

KulmanLab درج ذیل DXF اینٹیٹی اقسام پارس کرتا ہے:

| اینٹیٹی قسم | DXF کوڈ | نوٹس |
|-------------|----------|-------|
| لائن | `LINE` | |
| دائرہ | `CIRCLE` | |
| قوس | `ARC` | |
| بیضہ | `ELLIPSE` | |
| پولی لائن | `LWPOLYLINE` | |
| اسپلائن | `SPLINE` | |
| متن | `TEXT`، `MTEXT` | |
| ڈائمنشن | `DIMENSION` | |
| ملٹی لیڈر | `MULTILEADER` | |

لیئر ڈیفینیشنز اور لائن ٹائپ ٹیبلز بھی DXF فائل سے درآمد کیے جاتے ہیں جب موجود ہوں۔

غیر سپورٹ شدہ DXF اقسام استعمال کرنے والی اشیاء خاموشی سے چھوڑ دی جاتی ہیں — باقی ڈرائنگ بدستور لوڈ ہوتی ہے۔

## فائل نامکاری اور ذخیرہ

جب آپ فائل درآمد کرتے ہیں، فائل نام میں ٹائم اسٹیمپ شامل کیا جاتا ہے (مثلاً `myplan_May22_14:30:00.dxf`)۔ اس سے آپ [حالیہ فائلز](../files/) میں ایک ہی ڈرائنگ کے متعدد ورژن رکھ سکتے ہیں۔ اگر فائل نام میں پہلے سے ٹائم اسٹیمپ موجود ہے، تو وہی استعمال ہوتا ہے۔

ڈرائنگ درآمد کے بعد خود بخود براؤزر اسٹوریج (IndexedDB) میں محفوظ ہو جاتی ہے، اس لیے یہ [Files](../files/) پینل میں ظاہر ہوتی ہے اور صفحہ ریلوڈ کے بعد بھی باقی رہتی ہے۔

## موجودہ ڈرائنگ کا کیا ہوتا ہے

درآمد کرنے سے موجودہ کینوس کی جگہ نئی ڈرائنگ آ جاتی ہے۔ ضم یا الحاق کا کوئی آپشن نہیں ہے۔ اگر آپ کے پاس غیر محفوظ تبدیلیاں ہیں تو پہلے موجودہ ڈرائنگ [برآمد](../export/) کریں۔

## اسٹارٹ اپ پر

KulmanLab صفحہ لوڈ ہونے پر خود بخود آخری ترمیم شدہ فائل دوبارہ کھولتا ہے۔ اگر کوئی محفوظ فائل موجود نہ ہو، تو ایک ڈیفالٹ نمونہ ڈرائنگ لوڈ ہوتی ہے۔

## مسائل کا حل

| مسئلہ | ممکنہ وجہ | حل |
|---------|-------------|-----|
| درآمد کے بعد کینوس خالی ہے | DXF اشیاء غیر سپورٹ شدہ اقسام (مثلاً HATCH، INSERT) استعمال کرتی ہیں | اشیاء چھوڑ دی گئیں — ٹرمینل میں "no entities found" پیغام چیک کریں |
| Import بٹن کام نہیں کرتا | براؤزر نے فائل پِکر بلاک کیا | بٹن ایک بار پھر دبائیں؛ کچھ براؤزرز تازہ صارف اشارہ مانگتے ہیں |
| ڈائمنشنز غلط نظر آتے ہیں | DXF کسی ایسے ٹول سے جو غیر معیاری ڈائمنشن جیومیٹری لکھتا ہے | سورس ایپ سے موجودہ DXF ورژن استعمال کر کے دوبارہ برآمد کریں |
