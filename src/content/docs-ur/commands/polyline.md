# Polyline

`polyline` کمانڈ کسی بھی تعداد کے سیدھے سیگمنٹس کا متصل راستہ بناتی ہے، سب ایک `LWPOLYLINE` اینٹیٹی کے طور پر۔ کیونکہ پورا راستہ ایک آبجیکٹ ہے، اسے منتخب کرنے سے ہر سیگمنٹ منتخب ہو جاتا ہے — پوری شکل ایک آپریشن میں حرکت، گھماؤ، یا پیمانہ کریں۔ یہ زنجیری [Lines](./line) سے اہم فرق ہے جہاں ہر سیگمنٹ آزاد اینٹیٹی ہے۔

پولی لائنز **بند** بھی ہو سکتی ہیں: [Rectangle](./rectangle) کمانڈ وہی `LWPOLYLINE` اینٹیٹی کلوز فلیگ سیٹ کے ساتھ استعمال کرتی ہے۔

## پولی لائن بنانا

1. ٹرمینل میں `polyline` ٹائپ کریں یا **Polyline** ٹول بار بٹن پر کلک کریں۔
2. **پہلے نقطے پر کلک کریں**، یا درست کوآرڈینیٹ کے لیے `X,Y` ٹائپ کریں اور **Enter** دبائیں۔
3. **ہر اگلے نقطے پر کلک کریں** — ہر کلک ایک سیگمنٹ شامل کرتا ہے۔ کوآرڈینیٹ اندراج ہر مرحلے پر کام کرتا ہے۔
4. ختم کرنے کے لیے **Enter** یا **Space** دبائیں (کم از کم ۲ نقاط درکار)۔

```
  ●──────●
  1st    2nd
          \
           \  سیگمنٹ 3 (جاری — کرسر یہاں)
            ●  ← شامل کرنے کے لیے کلک، ختم کرنے کے لیے Enter/Space
```

**Escape** دبانے سے تمام رکھے گئے نقاط ضائع ہو جاتے ہیں اور کمانڈ ختم ہوتی ہے۔

## کوآرڈینیٹ اندراج

کلک کرنے کی بجائے، کسی بھی ورٹیکس کے لیے درست پوزیشن ٹائپ کریں:

1. X قدر ٹائپ کریں۔
2. `,` دبائیں — ٹرمینل `[X], [Y{cursor}]` دکھاتا ہے۔
3. Y قدر ٹائپ کریں۔
4. ورٹیکس رکھنے کے لیے **Enter** دبائیں۔

## زاویہ لاکنگ اور درست سیگمنٹ لمبائی

[Line](./line#angle-locking-and-exact-length-input) کمانڈ کی طرح وہی 45° اسنیپ منطق کسی بھی دو متتالی نقاط کے درمیان لاگو ہوتی ہے۔ جب محور پر لاک ہو:

| کلید | عمل |
|-----|--------|
| `0`–`9`, `.` | سیگمنٹ لمبائی میں ہندسہ شامل کریں |
| `-` | منفی لمبائی — محور کے ساتھ سمت پلٹائیں (صرف پہلا حرف) |
| `Backspace` | آخری ٹائپ کردہ حرف حذف کریں |
| `Enter` | ٹائپ کردہ فاصلے پر اگلا نقطہ رکھیں |

موجودہ جمع لمبائی ریئل ٹائم میں ٹرمینل پرامپٹ میں ظاہر ہوتی ہے۔

## کی بورڈ حوالہ

| کلید | عمل |
|-----|--------|
| `0`–`9`, `.`, `-` | X کوآرڈینیٹ اندراج شروع کریں، یا زاویہ لاک پر سیگمنٹ لمبائی |
| `,` | X لاک کریں اور Y اندراج پر جائیں |
| `Backspace` | آخری ٹائپ کردہ حرف حذف کریں |
| `Enter` | ٹائپ کردہ کوآرڈینیٹ یا لمبائی تصدیق کریں، یا پولی لائن ختم کریں اگر کوئی ان پٹ نہیں اور ≥ ۲ نقاط ہیں |
| `Space` | پولی لائن ختم کریں (Enter کی طرح جب کوئی ان پٹ جاری نہیں) |
| `Escape` | تمام نقاط ضائع کریں اور نکلیں |

## گرپ ترمیم — ورٹیکسیز اور سیگمنٹ درمیانی نقاط

منتخب پولی لائن دو قسم کے گرپس دکھاتی ہے:

| گرپ | پوزیشن | کیا کرتا ہے |
|------|----------|--------------|
| **ورٹیکس** | ہر رکھے گئے نقطے پر | اس ورٹیکس کو دوبارہ پوزیشن کرنے کے لیے ڈریگ کریں؛ تمام متصل سیگمنٹس پیچھے کھینچے جاتے ہیں |
| **سیگمنٹ درمیانی نقطہ** | ہر سیگمنٹ کا مرکز | اس سیگمنٹ کے **دونوں** اختتامی نقاط ایک ساتھ منتقل کرنے کے لیے ڈریگ کریں، سیگمنٹ کی لمبائی اور زاویہ برقرار |

سیگمنٹ درمیانی نقطہ گرپ پولی لائنز کے لیے منفرد ہے — یہ آپ کو انفرادی سیگمنٹ کو اس کی لمبائی تبدیل کیے بغیر اطراف میں سلائیڈ کرنے دیتا ہے۔

## پولی لائنز منتخب کرنا

| طریقہ | برتاؤ |
|--------|-----------|
| **کلک** | پولی لائن منتخب کرتا ہے اگر کلک کسی بھی سیگمنٹ کے ہٹ ٹیسٹ فاصلے کے اندر ہو |
| **دائیں ڈریگ** (سخت) | تمام ورٹیکسیز باکس کے اندر ہونی چاہیں |
| **بائیں ڈریگ** (کراسنگ) | کوئی بھی سیگمنٹ جو باکس کی سرحد کراس کرے پوری پولی لائن منتخب کرے |

## سپورٹ شدہ ترمیم کمانڈز

پولی لائنز تمام عمومی تبدیلیاں اور آفسیٹ سپورٹ کرتی ہیں، لیکن ٹرم یا ایکسٹینڈ **نہیں** (وہ صرف [Line](./line) کے لیے ہیں):

| کمانڈ | پولی لائن پر کیا ہوتا ہے |
|---------|------------------------------|
| [Move](./move) | تمام ورٹیکسیز ایک ہی بے گھری سے منتقل |
| [Copy](./copy) | نئی پوزیشن پر ایک جیسی پولی لائن بناتی ہے |
| [Rotate](./rotate) | تمام ورٹیکسیز چنے گئے بنیادی نقطے کے گرد گھماتی ہے |
| [Mirror](./mirror) | تمام ورٹیکسیز عکس محور پر منعکس کرتی ہے |
| [Scale](./scale) | تمام ورٹیکسیز بنیادی نقطے سے یکساں پیمانہ |
| [Offset](./offset) | مقررہ عمودی فاصلے پر متوازی پولی لائن بناتی ہے |
| [Delete](./delete) | پولی لائن ڈرائنگ سے ہٹاتی ہے |

## خصوصیات

جب پولی لائن منتخب ہو خصوصیات پینل دکھاتا ہے:

**عمومی**

| خاصیت | ڈیفالٹ | مطلب |
|----------|---------|---------|
| رنگ | 256 (ByLayer) | ACI رنگ انڈیکس |
| لیئر | `0` | لیئر تفویض |
| لائن ٹائپ | ByLayer | نامی لائن ٹائپ پیٹرن |
| لائن ٹائپ پیمانہ | 1 | لائن ٹائپ پیٹرن پر پیمانہ عنصر |
| موٹائی | 0 | اخراج موٹائی |

**جیومیٹری**

| خاصیت | مطلب |
|----------|---------|
| بند | آیا آخری ورٹیکس پہلے سے واپس جڑتی ہے |
| ورٹیکس تعداد | ورٹیکسیز کی کل تعداد |
| ورٹیکسیز | تمام ورٹیکسیز کی کوآرڈینیٹ فہرست |

## پولی لائن بمقابلہ لائن — کونسی کب استعمال کریں

| | پولی لائن | لائن |
|---|---------|------|
| اینٹیٹی تعداد | پورے راستے کے لیے ایک `LWPOLYLINE` | فی سیگمنٹ ایک `LINE` |
| بند شکل | ہاں (کلوز فلیگ) | نہیں |
| ٹرم / ایکسٹینڈ | نہیں | ہاں — سیگمنٹ بہ سیگمنٹ |
| سیگمنٹ درمیانی نقطہ گرپ | پورا سیگمنٹ منتقل کرتا ہے | اینٹیٹی کے لیے Move فعال کرتا ہے |
| بہترین | خاکے، خط، جو اشکال آپ پورا رکھیں | تعمیری لائنیں، جیومیٹری جو آپ ٹرم کریں گے |

## DXF — LWPOLYLINE اینٹیٹی

پولی لائنز DXF فائل میں `LWPOLYLINE` اشیاء کے طور پر محفوظ ہوتی ہیں۔ تمام خصوصیات — ورٹیکس کوآرڈینیٹس، کلوز فلیگ، رنگ، لیئر، لائن ٹائپ، لائن ٹائپ پیمانہ، اور موٹائی — بغیر نقصان کے راؤنڈ ٹرپ۔
