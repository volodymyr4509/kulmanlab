# Move

`move` کمانڈ منتخب اشیاء کو بنیادی نقطے سے منزل نقطے تک منتقل کرتی ہے۔ ہر منتخب اینٹیٹی پر لاگو بے گھری ویکٹر بنیاد سے منزل تک ہوتی ہے۔ منتقلی کے بعد تمام اشیاء نئی پوزیشن پر منتخب رہتی ہیں، مزید ترمیم کے لیے تیار۔

## شروع کرنے کے دو طریقے

**پہلے منتخب کریں، پھر حرکت کریں** — پہلے اشیاء منتخب کریں، پھر فعال کریں:

1. کینوس پر ایک یا زیادہ اشیاء منتخب کریں۔
2. ٹرمینل میں `move` ٹائپ کریں یا **Move** ٹول بار بٹن پر کلک کریں۔
3. **بنیادی نقطے پر کلک کریں**، یا درست کوآرڈینیٹ کے لیے `X,Y` ٹائپ کریں اور **Enter** دبائیں۔
4. **منزل پر کلک کریں** — تمام منتخب اشیاء بنیاد→منزل ویکٹر کے ذریعے حرکت کریں گی۔ کوآرڈینیٹ اندراج یہاں بھی کام کرتا ہے۔

**پہلے فعال کریں، پھر منتخب کریں** — بغیر کچھ منتخب کیے کمانڈ شروع کریں:

1. `move` ٹائپ کریں یا ٹول بار بٹن پر کلک کریں۔
2. **اشیاء منتخب کریں** — انفرادی اشیاء ٹوگل کرنے کے لیے کلک کریں، یا علاقے کے ذریعے منتخب کرنے کے لیے ڈریگ کریں۔
3. انتخاب تصدیق کرنے کے لیے **Enter** یا **Space** دبائیں۔
4. **بنیادی نقطے پر کلک کریں**، پھر **منزل پر کلک کریں** (دونوں مراحل پر کوآرڈینیٹ اندراج دستیاب ہے)۔

```
  پہلے:                    بعد:
  ● بنیاد                    → ● منزل
  [اینٹیٹی A]                   [اینٹیٹی A منتقل]
  [اینٹیٹی B]                   [اینٹیٹی B منتقل]
```

تمام منتخب اشیاء کا گھوسٹ پیش نظارہ بنیادی نقطے سے منزل تک کرسر کی پیروی کرتا ہے، کلک کرنے سے پہلے نتیجہ دکھاتا ہے۔

## کوآرڈینیٹ اندراج

بنیادی نقطہ یا منزل مرحلے پر، کلک کرنے کی بجائے درست پوزیشن ٹائپ کریں:

1. X قدر ٹائپ کریں۔
2. `,` دبائیں — ٹرمینل `[X], [Y{cursor}]` دکھاتا ہے۔
3. Y قدر ٹائپ کریں۔
4. تصدیق کے لیے **Enter** دبائیں۔

## زاویہ لاکنگ اور درست فاصلہ

بنیادی نقطہ سیٹ ہونے کے بعد کمانڈ 45° اسنیپ محور (0°، 45°، 90°، 135°، …) کی تلاش کرتی ہے۔ جب کرسر بنیاد سے کافی دور اور محور کے ایک گرپ چوڑائی کے اندر ہو تو سمت **لاک** ہو جاتی ہے۔ لاک ہونے پر:

- گھوسٹ پیش نظارہ محور سے اسنیپ کرتا ہے۔
- لاک سمت کے ساتھ بالکل اتنی دور منتقل کرنے کے لیے فاصلہ ٹائپ کریں اور **Enter** دبائیں۔
- کلک محور پر پروجیکٹ ہوتا ہے، تاکہ منزل ہمیشہ بالکل اس پر ہو۔

| کلید | عمل |
|-----|--------|
| `0`–`9`, `.` | فاصلہ قدر میں شامل کریں |
| `-` | منفی فاصلہ — محور کے ساتھ سمت پلٹائیں (صرف پہلا حرف) |
| `Backspace` | آخری ٹائپ کردہ حرف حذف کریں |
| `Enter` | ٹائپ کردہ فاصلے پر منتقلی لاگو کریں |

## کی بورڈ حوالہ

| کلید | عمل |
|-----|--------|
| `Enter` / `Space` | انتخاب تصدیق کریں اور بنیادی نقطہ مرحلے پر جائیں |
| `0`–`9`, `.`, `-` | X کوآرڈینیٹ اندراج شروع کریں، یا زاویہ لاک پر فاصلہ |
| `,` | X لاک کریں اور Y اندراج پر جائیں |
| `Backspace` | آخری ٹائپ کردہ حرف حذف کریں |
| `Enter` | کوآرڈینیٹ تصدیق کریں یا ٹائپ کردہ فاصلے پر منتقلی لاگو کریں |
| `Escape` | منسوخ کریں اور ری سیٹ کریں |

## گرپ سے Move فعال کرنا

منتخب [Line](../line/) کے **درمیانی نقطے کے گرپ** پر کلک کرنے سے Move خود بخود شروع ہوتا ہے، درمیانی نقطہ بنیادی نقطے کے طور پر پہلے سے سیٹ اور منتقلی مرحلہ فعال۔ یہ انتخاب مرحلے سے گزرے بغیر ایک لائن دوبارہ پوزیشن کرنے کا سب سے تیز طریقہ ہے۔

## کمانڈ کے دوران انتخاب

جب کمانڈ انتخاب مرحلے میں شروع ہو:

| طریقہ | برتاؤ |
|--------|-----------|
| **کلک** | کرسر کے نیچے اینٹیٹی انتخاب میں/سے ٹوگل کرتا ہے |
| **دائیں ڈریگ** (سخت) | باکس کے مکمل اندر موجود اشیاء شامل کرتا ہے |
| **بائیں ڈریگ** (کراسنگ) | باکس کی سرحد کو چھونے والی اشیاء شامل کرتا ہے |
| **Enter** / **Space** | انتخاب تصدیق کرتا ہے اور بنیادی نقطہ مرحلے پر جاتا ہے |

## منتقلی کے بعد

منتقل اشیاء نئی پوزیشن پر منتخب رہتی ہیں۔ اس کا مطلب آپ فوری طور پر:
- انہیں مزید آگے بڑھانے کے لیے **Move** دوبارہ چلا سکتے ہیں۔
- دوبارہ منتخب کیے بغیر [Copy](../copy/)، [Rotate](../rotate/)، یا [Scale](../scale/) چلا سکتے ہیں۔
- انہیں ہٹانے کے لیے **Delete** دبا سکتے ہیں۔

## Move بمقابلہ Copy

| | Move | Copy |
|---|------|------|
| اصل پوزیشن | خالی — اشیاء وہاں نہیں رہتیں | برقرار — اصل جگہ پر رہتی ہیں |
| نتیجے کی تعداد | اتنی ہی اشیاء | فی آپریشن ایک اضافی سیٹ |
| بعد میں انتخاب | نئی پوزیشن پر منتقل شدہ اشیاء منتخب | نئی پوزیشن پر کاپی شدہ اشیاء منتخب |
| بہترین | جیومیٹری دوبارہ پوزیشن کرنا | جیومیٹری نقل کرنا |

## سپورٹ شدہ اشیاء

Move ہر اینٹیٹی قسم پر کام کرتی ہے: Line، Polyline، Rectangle، Circle، Arc، Ellipse، Text، Spline، Dimension، Leader، اور دیگر سبھی۔ تمام اشیاء `translate(dx, dy)` لاگو کرتی ہیں اس لیے کوئی بھی خارج نہیں ہے۔
