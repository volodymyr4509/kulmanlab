# Files

**Files** کمانڈ **حالیہ فائلیں** پینل کھولتی ہے — آپ کے براؤزر کے مقامی اسٹوریج میں محفوظ تمام ڈرائنگز کی فہرست۔ پچھلی ڈرائنگ دوبارہ کھولنے یا وہ فائلیں ہٹانے کے لیے استعمال کریں جن کی ضرورت نہیں ہے۔

## حالیہ فائلیں پینل کیسے کھولیں

اسکرین کے اوپر File پینل میں **Files** ٹول بار بٹن (تاریخ آئیکن) پر کلک کریں۔ پینل کینوس کے بائیں طرف کھلتا ہے۔

## حالیہ فائلیں پینل

فہرست کی ہر اندراج فائل کا نام اور دو ایکشن بٹن دکھاتی ہے:

| بٹن | عمل |
|--------|--------|
| ✓ **Restore** | ڈرائنگ کینوس پر لوڈ کریں — موجودہ مواد تبدیل کرتا ہے |
| ✕ **Delete** | براؤزر اسٹوریج سے فائل مستقل طور پر ہٹائیں |

اگر ابھی تک کوئی فائل محفوظ نہیں ہوئی، پینل "No files saved" دکھاتا ہے۔

## فائلیں کیسے محفوظ ہوتی ہیں

KulmanLab ڈرائنگز **IndexedDB** میں محفوظ کرتا ہے — ایک مستقل ان براؤزر ڈیٹابیس۔ اس کا مطلب ہے:

- فائلیں صفحہ ری لوڈ اور براؤزر دوبارہ شروع ہونے پر بھی رہتی ہیں۔
- فائلیں **مقامی طور پر آپ کے ڈیوائس پر** محفوظ ہوتی ہیں — کسی سرور پر اپلوڈ نہیں ہوتیں۔
- ہر براؤزر اور ڈیوائس کی اپنی آزاد اسٹوریج ہوتی ہے۔ ایک کمپیوٹر پر Chrome میں محفوظ ڈرائنگ Firefox یا دوسری مشین پر دستیاب نہیں ہوگی۔
- براؤزر کا سائٹ ڈیٹا صاف کرنے سے تمام محفوظ فائلیں مٹ جائیں گی۔

## اسٹارٹ اپ پر خودکار فائل لوڈنگ

جب آپ KulmanLab CAD کھولتے ہیں، ایپ خود بخود اسٹوریج سے **سب سے حال ہی میں ترمیم کردہ فائل** لوڈ کرتی ہے۔ آپ کو ہر بار Files پینل سے دستی طور پر کھولنے کی ضرورت نہیں۔

## اسٹوریج منظم کرنا

آپ جتنی ڈرائنگز محفوظ کر سکتے ہیں اس کی کوئی مقررہ حد نہیں، لیکن براؤزر اسٹوریج محدود ہے۔ اگر آپ کو اسٹوریج انتباہات نظر آئیں، Files پینل سے پرانی فائلیں حذف کریں۔

ایک ہی وقت میں تمام محفوظ ڈرائنگز ہٹانے کے لیے، [WipeStorage](./wipestorage) کمانڈ استعمال کریں۔

## فائل کے نام اور ورژننگ

ہر فائل کے نام میں ٹائم اسٹیمپ ہوتا ہے (مثلاً `myplan_May22_14:30:00.dxf`)۔ یہ ٹائم اسٹیمپ [import](./import) کرتے وقت خود بخود شامل ہوتا ہے، یہ یقینی بناتے ہوئے کہ ایک ہی سورس فائل دوبارہ درآمد کرنے سے پچھلے ورژن کو اوور رائٹ کرنے کی بجائے نئی اندراج بنتی ہے۔

## متعلقہ کمانڈز
