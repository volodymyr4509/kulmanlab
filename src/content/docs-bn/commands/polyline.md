# Polyline

`polyline` কমান্ড একটি একক `LWPOLYLINE` এন্টিটি হিসেবে সংরক্ষিত যেকোনো সংখ্যক সরল সেগমেন্টের একটি সংযুক্ত পথ আঁকে। পুরো পথটি একটি অবজেক্ট হওয়ায়, এটি নির্বাচন করলে প্রতিটি সেগমেন্ট একসাথে নির্বাচিত হয় — একক অপারেশনে পুরো আকারটি সরান, ঘোরান বা স্কেল করুন। এটিই চেইন করা [Lines](../line/) থেকে মূল পার্থক্য, যেখানে প্রতিটি সেগমেন্ট একটি স্বাধীন এন্টিটি।

পলিলাইনগুলি **বদ্ধ** হতে পারে: [Rectangle](../rectangle/) কমান্ড একটি ক্লোজ ফ্ল্যাগ সেট সহ একই `LWPOLYLINE` এন্টিটি ব্যবহার করে।

## একটি পলিলাইন আঁকা

1. টার্মিনালে `polyline` টাইপ করুন বা **Polyline** টুলবার বাটনে ক্লিক করুন।
2. **প্রথম পয়েন্টে ক্লিক করুন**, বা সঠিক কোঅর্ডিনেটের জন্য `X,Y` টাইপ করুন এবং **Enter** চাপুন।
3. **প্রতিটি পরবর্তী পয়েন্টে ক্লিক করুন** — প্রতিটি ক্লিক একটি সেগমেন্ট যোগ করে। প্রতিটি ধাপে কোঅর্ডিনেট এন্ট্রি কাজ করে।
4. শেষ করতে **Enter** বা **Space** চাপুন (কমপক্ষে ২টি পয়েন্ট প্রয়োজন)।

```
  ●──────●
  ১ম     ২য়
          \
           \  সেগমেন্ট ৩ (চলছে — কার্সর এখানে)
            ●  ← যোগ করতে ক্লিক করুন, শেষ করতে Enter/Space
```

যেকোনো সময় **Escape** চাপলে সমস্ত স্থাপিত পয়েন্ট বাদ দেওয়া হয় এবং কমান্ড থেকে বের হওয়া হয়।

## কোঅর্ডিনেট এন্ট্রি

ক্লিক করার বদলে, যেকোনো ভার্টেক্সের জন্য একটি সঠিক পজিশন টাইপ করুন:

1. X মান টাইপ করুন।
2. `,` চাপুন — টার্মিনাল `[X], [Y{কার্সর}]` দেখায়।
3. Y মান টাইপ করুন।
4. ভার্টেক্স স্থাপন করতে **Enter** চাপুন।

## কোণ লকিং এবং সঠিক সেগমেন্ট দৈর্ঘ্য

[Line](../line/#angle-locking-and-exact-length-input) কমান্ডের মতো একই ৪৫° স্ন্যাপ লজিক যেকোনো দুটি ধারাবাহিক পয়েন্টের মধ্যে প্রযোজ্য। একটি অক্ষে লক থাকলে:

| কী | ক্রিয়া |
|-----|--------|
| `0`–`9`, `.` | সেগমেন্টের দৈর্ঘ্যে সংখ্যা যোগ করুন |
| `-` | নেতিবাচক দৈর্ঘ্য — অক্ষ বরাবর দিক বিপরীত করে (শুধু প্রথম অক্ষর) |
| `Backspace` | শেষ টাইপ করা অক্ষর মুছুন |
| `Enter` | টাইপ করা দূরত্বে পরবর্তী পয়েন্ট স্থাপন করুন |

বর্তমান সংগৃহীত দৈর্ঘ্য রিয়েল টাইমে টার্মিনাল প্রম্পটে দেখা যায়। লক থাকাকালীন ক্লিক করলে অক্ষে প্রজেক্ট হয় তাই নতুন ভার্টেক্স ঠিক সেই অক্ষে পড়ে।

## কীবোর্ড রেফারেন্স

| কী | ক্রিয়া |
|-----|--------|
| `0`–`9`, `.`, `-` | X কোঅর্ডিনেট এন্ট্রি শুরু করুন, বা কোণ-লক থাকলে সেগমেন্ট দৈর্ঘ্য |
| `,` | X লক করুন এবং Y এন্ট্রিতে যান |
| `Backspace` | শেষ টাইপ করা অক্ষর মুছুন |
| `Enter` | টাইপ করা কোঅর্ডিনেট বা দৈর্ঘ্য নিশ্চিত করুন, বা কিছু টাইপ না করলে এবং ≥ ২ পয়েন্ট থাকলে পলিলাইন শেষ করুন |
| `Space` | পলিলাইন শেষ করুন (কোনো ইনপুট চলছে না থাকলে Enter এর মতো) |
| `Escape` | সমস্ত পয়েন্ট বাদ দিন এবং বের হন |

## গ্রিপ সম্পাদনা — ভার্টেক্স এবং সেগমেন্ট মিডপয়েন্ট

একটি নির্বাচিত পলিলাইন দুই ধরনের গ্রিপ দেখায়:

| গ্রিপ | পজিশন | কী করে |
|------|----------|--------------|
| **ভার্টেক্স** | প্রতিটি স্থাপিত পয়েন্টে | সেই ভার্টেক্স পুনরায় স্থাপন করতে টেনে আনুন; সব সংযুক্ত সেগমেন্ট অনুসরণ করে প্রসারিত হয় |
| **সেগমেন্ট মিডপয়েন্ট** | প্রতিটি সেগমেন্টের কেন্দ্রে | সেগমেন্টের **উভয়** শেষ পয়েন্ট একসাথে স্থানান্তর করতে টেনে আনুন, সেগমেন্টের দৈর্ঘ্য ও কোণ অক্ষুণ্ণ রেখে |

সেগমেন্ট-মিডপয়েন্ট গ্রিপ পলিলাইনের জন্য অনন্য — এটি দৈর্ঘ্য না বদলে পৃথক সেগমেন্ট পাশে সরাতে দেয়। একটি [Line](../line/) এ, মিডপয়েন্ট গ্রিপ পরিবর্তে পুরো এন্টিটির জন্য Move কমান্ড সক্রিয় করে।

পুরো পলিলাইন সরানোর একক "পুরো পলিলাইন সরান" গ্রিপ নেই। পুরো পথ সরাতে [Move](../move/) কমান্ড ব্যবহার করুন।

## পলিলাইন নির্বাচন

| পদ্ধতি | আচরণ |
|--------|-----------|
| **ক্লিক** | যেকোনো সেগমেন্টের হিট-টেস্ট দূরত্বের মধ্যে ক্লিক পড়লে পলিলাইন নির্বাচিত হয় |
| **ডানে টানুন** (কঠোর) | সমস্ত ভার্টেক্স বাক্সের ভেতরে পড়তে হবে |
| **বামে টানুন** (ক্রসিং) | বাক্সের সীমানা অতিক্রমকারী যেকোনো সেগমেন্ট পুরো পলিলাইন নির্বাচিত করে |

পলিলাইন একটি এন্টিটি হওয়ায়, যেকোনো সেগমেন্ট স্পর্শকারী ক্রসিং নির্বাচন সমস্ত সেগমেন্ট নির্বাচিত করে।

## সমর্থিত সম্পাদনা কমান্ড

পলিলাইন সমস্ত সাধারণ রূপান্তর এবং অফসেট সমর্থন করে, কিন্তু ট্রিম বা এক্সটেন্ড নয় (সেগুলি [Line](../line/)-শুধু):

| কমান্ড | পলিলাইনে কী হয় |
|---------|------------------------------|
| [Move](../move/) | সমস্ত ভার্টেক্স একই বিচ্যুতিতে স্থানান্তরিত করে |
| [Copy](../copy/) | নতুন পজিশনে একটি অভিন্ন পলিলাইন তৈরি করে |
| [Rotate](../rotate/) | বেছে নেওয়া বেস পয়েন্টের চারপাশে সমস্ত ভার্টেক্স ঘোরায় |
| [Mirror](../mirror/) | মিরর অক্ষ জুড়ে সমস্ত ভার্টেক্স প্রতিফলিত করে |
| [Scale](../scale/) | বেস পয়েন্ট থেকে সমস্ত ভার্টেক্স সমানভাবে স্কেল করে |
| [Offset](../offset/) | নির্দিষ্ট লম্ব দূরত্বে একটি সমান্তরাল পলিলাইন তৈরি করে |
| [Delete](../delete/) | ড্রয়িং থেকে পলিলাইন সরায় |

## প্রপার্টি

একটি পলিলাইন নির্বাচিত হলে প্রপার্টি প্যানেল দেখায়:

**সাধারণ**

| প্রপার্টি | ডিফল্ট | অর্থ |
|----------|---------|---------|
| Color | 256 (ByLayer) | ACI রঙের সূচক |
| Layer | `0` | লেয়ার অ্যাসাইনমেন্ট |
| Linetype | ByLayer | নামযুক্ত লাইনটাইপ প্যাটার্ন |
| Linetype Scale | 1 | লাইনটাইপ প্যাটার্নের স্কেল ফ্যাক্টর |
| Thickness | 0 | এক্সট্রুশন পুরুত্ব |

**জ্যামিতি**

| প্রপার্টি | অর্থ |
|----------|---------|
| Closed | শেষ ভার্টেক্স প্রথমের সাথে সংযুক্ত কিনা |
| Vertex Count | মোট ভার্টেক্সের সংখ্যা |
| Vertices | সমস্ত ভার্টেক্সের কোঅর্ডিনেট তালিকা |

## Polyline বনাম Line — কোনটি ব্যবহার করবেন

| | Polyline | Line |
|---|---------|------|
| এন্টিটির সংখ্যা | সম্পূর্ণ পথের জন্য একটি `LWPOLYLINE` | প্রতি সেগমেন্টে একটি `LINE` |
| বদ্ধ আকার | হ্যাঁ (ক্লোজ ফ্ল্যাগ) | না |
| Trim / Extend | না | হ্যাঁ — সেগমেন্ট অনুযায়ী |
| সেগমেন্ট-মিডপয়েন্ট গ্রিপ | পুরো সেগমেন্ট স্থানান্তরিত করে | এন্টিটির জন্য Move সক্রিয় করে |
| সেরা জন্য | আউটলাইন, কনট্যুর, সম্পূর্ণ রাখা আকার | কনস্ট্রাকশন লাইন, ট্রিম করার জ্যামিতি |

## DXF — LWPOLYLINE এন্টিটি

পলিলাইনগুলি DXF ফাইলে `LWPOLYLINE` এন্টিটি হিসেবে সেভ হয়। সমস্ত প্রপার্টি — ভার্টেক্স কোঅর্ডিনেট, ক্লোজড ফ্ল্যাগ, রঙ, লেয়ার, লাইনটাইপ, লাইনটাইপ স্কেল এবং পুরুত্ব — ক্ষতি ছাড়াই রাউন্ড-ট্রিপ করে। [Rectangle](../rectangle/) কমান্ড দিয়ে আঁকা আয়তক্ষেত্রগুলিও `LWPOLYLINE` (বদ্ধ, চারটি ভার্টেক্স) হিসেবে সেভ হয় এবং DXF স্তরে অপরিচিত।

যেকোনো DXF-সামঞ্জস্যপূর্ণ অ্যাপ্লিকেশন (LibreCAD, FreeCAD, ইত্যাদি) থেকে `LWPOLYLINE` এন্টিটিগুলি এডিটরে সম্পূর্ণরূপে সম্পাদনযোগ্য পলিলাইন হিসেবে পড়া হয়।
