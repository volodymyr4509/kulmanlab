# Import

Команда **Import** завантажує існуючий кресленик з вашої локальної файлової системи в KulmanLab CAD. Підтримуються як стандартний формат **DXF**, так і власний формат **JSON** KulmanLab.

## Як імпортувати файл

1. Натисніть кнопку **Import** на панелі інструментів (іконка папки) у панелі File у верхній частині екрана.
2. Відкривається вибір файлів браузера. Перейдіть до вашого файлу кресленика та виберіть його.
3. Кресленик завантажується на полотно негайно. Область перегляду автоматично підганяє всі об'єкти.

Крім того, можна перетягнути файл безпосередньо на полотно.

## Підтримувані формати файлів

| Формат | Розширення | Коли використовувати |
|--------|-----------|----------------------|
| **DXF** | `.dxf` | Кресленики з AutoCAD, FreeCAD, LibreCAD або інших CAD інструментів |
| **JSON** *(рідний)* | `.json` | Кресленики раніше збережені з KulmanLab CAD — повна точність |

## Що імпортується з DXF

KulmanLab розбирає такі типи об'єктів DXF:

| Тип об'єкта | DXF код | Примітки |
|-------------|---------|----------|
| Лінія | `LINE` | |
| Коло | `CIRCLE` | |
| Дуга | `ARC` | |
| Еліпс | `ELLIPSE` | |
| Полілінія | `LWPOLYLINE` | |
| Сплайн | `SPLINE` | |
| Текст | `TEXT`, `MTEXT` | |
| Розмір | `DIMENSION` | |
| Мультивиноска | `MULTILEADER` | |

Визначення шарів та таблиці типів ліній також імпортуються з файлу DXF якщо присутні.

Об'єкти що використовують непідтримувані типи DXF тихо пропускаються — решта кресленика все одно завантажується.

## Іменування файлів та зберігання

При імпорті файлу, до назви файлу додається мітка часу (наприклад `myplan_May22_14:30:00.dxf`). Це дозволяє зберігати кілька версій одного кресленика у [Recent Files](../files/) без конфліктів назв. Якщо назва файлу вже містить мітку часу, вона використовується як є.

Кресленик автоматично зберігається у сховище браузера (IndexedDB) після імпорту, тому він з'являється у панелі [Files](../files/) та виживає перезавантаження сторінки.

## Що відбувається з поточним кресленником

Імпорт замінює поточне полотно. Немає злиття або доповнення. Якщо є незбережені зміни, спочатку [експортуйте](../export/) поточний кресленик.

## При запуску

KulmanLab автоматично повторно відкриває останній відредагований файл при завантаженні сторінки. Якщо збережених файлів немає, завантажується типовий демонстраційний кресленик.

## Усунення несправностей

| Проблема | Ймовірна причина | Вирішення |
|---------|-----------------|----------|
| Полотно порожнє після імпорту | Об'єкти DXF використовують непідтримувані типи (наприклад HATCH, INSERT) | Об'єкти були пропущені — перевірте повідомлення "no entities found" у терміналі |
| Кнопка Import не реагує | Браузер заблокував вибір файлів | Натисніть кнопку ще раз; деякі браузери вимагають нової дії користувача |
| Розміри виглядають неправильно | DXF з інструменту що записує нестандартну геометрію розмірів | Повторно експортуйте з вихідної програми використовуючи поточну версію DXF |

## Пов'язані команди

- [Export](../export/) — завантажити поточний кресленик як DXF або JSON

- [New File](../new-file/) — почати порожній кресленик
