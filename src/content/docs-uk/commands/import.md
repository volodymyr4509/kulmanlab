# Import

Команда `import` відкриває файл кресленика з вашого пристрою та завантажує його в редактор. Підтримуються формати DXF та власний JSON формат KulmanLab CAD.

## Два способи імпорту

- **Кнопка панелі інструментів / Термінал**: натисніть **Import** або введіть `import` — відкривається стандартний діалог вибору файлу.
- **Перетягнути і відпустити**: перетягніть файл `.dxf` або `.json` безпосередньо на полотно — файл завантажується негайно.

## Що імпортується з DXF

| Об'єкт DXF | Код | Імпортується? |
|-----------|-----|---------------|
| LINE | `LINE` | Так |
| Полілінія | `LWPOLYLINE` | Так |
| Коло | `CIRCLE` | Так |
| Дуга | `ARC` | Так |
| Еліпс | `ELLIPSE` | Так |
| Сплайн | `SPLINE` | Так |
| Текст | `TEXT` / `MTEXT` | Так |
| Розміри | `DIMENSION` | Так |
| Виноска | `MLEADER` | Так |
| 3D тіла, штрихування, блоки | Різні | Ні — ігноруються |

## Іменування та зберігання файлів

При імпорті до назви файлу **автоматично додається мітка часу** — той самий файл можна імпортувати кілька разів без перезапису:

```
schematic.dxf → schematic_2026-01-15_14:32:07
```

Файл зберігається у **IndexedDB** браузера — локально на цьому пристрої та браузері.

## Що відбувається з поточним кресленником

Поточний кресленик **замінюється** — не об'єднується. Збережіть поточний кресленик через [Export](./export) перед імпортом, якщо потрібно зберегти роботу.

## При запуску

При відкритті KulmanLab CAD автоматично завантажується **найновіший збережений кресленик** з IndexedDB. Жодних додаткових дій не потрібно.

## Усунення несправностей

| Проблема | Рішення |
|---------|---------|
| Файл не відкривається | Перевірте що файл `.dxf` або `.json`, не пошкоджений |
| Деякі об'єкти відсутні | Вони можуть не підтримуватися (3D тіла, блоки) — нормальна поведінка |
| Кресленик виглядає порожнім | Спробуйте [Fit](./fit) — об'єкти можуть бути за межами видимого |
| Помилка великого файлу | Дуже великі DXF (1000+ об'єктів) займають кілька секунд |
