# Undo

Команда `undo` скасовує останню зміну в кресленику — один крок за одне виконання. Кожне додавання, видалення або редагування об'єктів записується як окремий запис в історії. Undo рухається через ці записи у зворотньому порядку.

## Як скасувати

- Введіть `undo` у терміналі, або
- Натисніть **Ctrl+Z** (Windows/Linux) або **Cmd+Z** (Mac), або
- Натисніть кнопку **Undo** на панелі інструментів.

Кожне виконання скасовує одну записану дію. Виконуйте повторно для переходу далі назад в часі.

## Поведінка історії

| Деталь | Значення |
|--------|----------|
| Кроків на файл | До **20** |
| Сховище | Браузер (IndexedDB / localStorage), на кожен файл |
| Виживає перезавантаження сторінки | Так — історія відновлюється при повторному відкритті файлу |
| Нова дія після Undo | Очищає всі записи Redo попереду поточної позиції |
| Найстаріший запис при переповненні | Видаляється, щоб звільнити місце для нових змін |

Кожна зміна об'єктів записується: малювання нових об'єктів, видалення об'єктів, редагування ручками кінцевих точок, застосування Move, Rotate, Scale, Mirror, Trim, Extend та Offset — всі вони створюють записи в історії.

## Що можна скасувати

- Малювання об'єктів (Line, Polyline, Circle, Arc тощо)
- Редагування об'єктів (Move, Copy, Rotate, Scale, Delete тощо)
- Редагування ручками
- Зміни властивостей
- Операції Trim, Extend, Offset

## Undo проти Redo

| | Undo | Redo |
|---|------|------|
| Напрямок | Рухається **назад** по історії | Рухається **вперед** по скасованих записах |
| Доступний коли | Існує щонайменше одна записана дія | Було виконано щонайменше одне Undo і не виконано нових дій |
| Очищується | Нічим — накопичується до ліміту 20 кроків | Будь-якою новою дією малювання |

Використовуйте [Redo](../redo/) для повторного застосування скасованої дії. Кнопки панелі інструментів стають сірими, коли відповідний напрямок недоступний.
