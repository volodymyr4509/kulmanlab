# Redo

Команда `redo` рухається вперед по історії Undo, повторно застосовуючи дії, скасовані командою [Undo](../undo/). Redo доступний лише тоді, коли ви відступили назад за допомогою Undo і ще не виконали нових змін.

## Як використовувати

- Введіть `redo` у терміналі, або
- Натисніть **Ctrl+Y** (Windows/Linux) або **Cmd+Y** (Mac), або
- Натисніть кнопку **Redo** на панелі інструментів.

Кожне виконання повторно застосовує одну раніше скасовану дію. Виконуйте повторно для переходу вперед по всіх доступних записах Redo.

## Поведінка стеку Redo

| Деталь | Поведінка |
|--------|-----------|
| Доступний після | Одного або більше кроків [Undo](../undo/) |
| Очищується | **Будь-якою новою дією малювання** — додаванням, редагуванням або видаленням об'єкта |
| Сховище | Браузер, на кожен файл — виживає перезавантаження сторінки (доки нова дія не була виконана до перезавантаження) |
| Максимальна глибина | До 20 записів (той самий пул що й Undo) |

Після того як новий об'єкт намальовано, видалено або змінено, стек Redo очищається і ці записи не можна відновити. Повторно можна застосувати лише скасовані дії, які не були замінені новою роботою.

## Redo проти Undo

| | Redo | Undo |
|---|------|------|
| Напрямок | Рухається **вперед** по скасованих записах | Рухається **назад** по історії |
| Доступний коли | Після щонайменше одного Undo, без нових виконаних дій | Існує щонайменше одна записана дія |
| Очищується | Будь-якою новою дією малювання | Нічим |

Кнопка Redo на панелі інструментів стає сірою, коли немає записів для повторення. Спочатку скористайтеся командою [Undo](../undo/), щоб створити записи Redo.
