# Redo

Команда `redo` повторює останню дію, скасовану командою [Undo](./undo). Якщо після Undo ви внесли нові зміни, стек Redo очищається — нові зміни замінюють скасовану гілку.

## Як використовувати

- Введіть `redo` у терміналі, або
- Натисніть **Ctrl+Y** (Windows/Linux) або **Cmd+Y** (Mac), або
- Натисніть кнопку **Redo** на панелі інструментів.

## Поведінка стеку Redo

| Ситуація | Стек Redo |
|---------|-----------|
| Після Undo | Доступний — містить скасовані кроки |
| Після виконання нової дії | Очищується — нові зміни замінюють гілку |
| Після Redo до кінця | Порожній — немає більше кроків для повторення |
| Зберігання | В пам'яті (не в IndexedDB) |
| Максимальна глибина | 20 кроків |

## Redo проти Undo

| | Redo | Undo |
|---|------|------|
| Напрямок | Вперед в історії | Назад в історії |
| Ярлик | Ctrl+Y / Cmd+Y | Ctrl+Z / Cmd+Z |
| Очищується коли | Нова дія після Undo | Ніколи (завжди можна Undo) |
