# Files

Команда `files` відкриває список кресленників, збережених у локальному сховищі браузера (IndexedDB). Звідси можна відновити, видалити або завантажити будь-який кресленик.

## Відкриття панелі Files

Натисніть кнопку **Files** на панелі інструментів або введіть `files` у терміналі. Відображається список збережених кресленників з назвами та датами.

## Кнопки для кожного кресленника

| Кнопка | Дія |
|--------|-----|
| **Restore** | Завантажує кресленик у редактор (замінює поточний) |
| **Delete** | Видаляє кресленик зі сховища (незворотно) |

## Стан "Немає файлів"

Якщо жодного кресленника не збережено — відображається повідомлення "Немає збережених файлів". Виконайте будь-яку дію на кресленику або імпортуйте файл — він з'явиться у списку.

## Сховище IndexedDB

| Характеристика | Деталі |
|----------------|--------|
| **Тип сховища** | IndexedDB (локально у браузері) |
| **Видимість** | Лише на цьому пристрої та браузері |
| **Очищення сайту** | Видалення даних сайту очищує всі кресленники |
| **Обмеження розміру** | Немає фіксованого ліміту, але кінцева місткість пристрою |

Кресленники **не** синхронізуються між браузерами або пристроями. Для перенесення між пристроями використовуйте [Export](../export/) → JSON → [Import](../import/).

## Автоматичне завантаження при запуску

При відкритті KulmanLab CAD **найновіший збережений кресленик** завантажується автоматично. Якщо сховище порожнє — відкривається новий порожній кресленик.

## Іменування та версіонування файлів

При [імпорті](../import/) DXF-файлу до назви додається **мітка часу**, тому той самий файл можна імпортувати кілька разів без перезапису попередніх версій. Наприклад: `schematic_2026-01-15_14:32:07`.

## Пов'язані команди

- [Import](../import/) — відкрити файл з пристрою
- [Export](../export/) — завантажити кресленик на пристрій
- [Wipe Storage](../wipestorage/) — видалити всі кресленики зі сховища
- [New File](../new-file/) — почати новий порожній кресленик
